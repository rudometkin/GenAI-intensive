# PROJECTCLONE — YouTube Social Media Listener

## Какой сервис клонировался и почему

Клонировался класс инструментов **social media listening** — конкретно функциональность таких сервисов как **Brandwatch**, **Mention.com**, **YouScan**, но сужено до YouTube.

Эти сервисы собирают упоминания бренда, продукта или темы на разных платформах, структурируют их и выделяют боли/тренды аудитории. Стоимость — от $500/мес и выше. Выбрал YouTube, потому что:
- Комментарии и транскрипты публичны и доступны без платного API
- YouTube Shorts — высококонкурентная зона, где авторы/бренды нуждаются в быстрой обратной связи от аудитории
- Можно собрать минимальный рабочий инструмент за один день без бэкенда

## Навыки, которые потребовались

| Навык | Зачем |
|---|---|
| Python scripting | основа всей логики |
| работа с CLI-инструментами (yt-dlp) | получение метаданных видео |
| работа со сторонними API без ключей | youtube-comment-downloader обходит ограничения |
| Google Sheets API + сервисный аккаунт | вывод данных без UI |
| базовый NLP / keyword scoring | детектирование болей без LLM |
| отладка breaking changes в библиотеках | несколько библиотек меняли API между версиями |

## Продуктовые модули и покрытие скиллами

```
YouTube Social Listening
├── [M1] Источник данных
│   ├── метаданные видео         → yt-dlp (CLI, subprocess)
│   ├── транскрипты              → youtube-transcript-api
│   └── комментарии              → youtube-comment-downloader
│
├── [M2] Анализ
│   └── pain points summary      → keyword scoring (локально, без LLM)
│
└── [M3] Хранение / вывод
    └── Google Sheets            → gspread + Service Account
```

Каждый модуль закрыт одной библиотекой. Никакого бэкенда — скрипт запускается локально.

## Код

`yt_listener.py` — единственный файл. Запуск:

```bash
pip install gspread google-auth youtube-comment-downloader youtube-transcript-api yt-dlp
python yt_listener.py
```

Конфиг в начале файла:
- `CREDS_PATH` — путь к JSON сервисного аккаунта Google
- `SHEET_ID` — ID Google Sheets таблицы
- `VIDEOS` — список URL видео (поддерживает Shorts и обычные видео)

## Логика создания

**Как создавалось:**
1. Определили цель — вытащить из YouTube всё, что нужно для анализа аудитории, и положить в таблицу
2. Нашли библиотеки под каждый тип данных (метаданные, транскрипт, комментарии)
3. Подключили Google Sheets как хранилище без UI
4. Добавили простой анализ болей через keyword scoring

**Что было самым сложным:**
- YouTube активно блокирует скачивание аудио через yt-dlp (403 ошибки) — пришлось отказаться от Whisper и переключиться на встроенные субтитры YouTube
- API библиотек менялись между версиями — имена методов и классов не совпадали с документацией, фиксили вручную

**Что осталось открытым:**
- Pain points summary сейчас работает на keyword matching — поверхностно, много ложных срабатываний
- Нет поддержки каналов целиком (только отдельные URL)
- Нет расписания / автозапуска
- Нет дедупликации (при повторном запуске таблица перезаписывается полностью)

## ⭐ Как построить ров вокруг этого продукта

### 1. Ров данных (Data Moat)
Накапливать историю комментариев по нишам и каналам — через 3–6 месяцев появляется датасет, которого ни у кого нет. Конкурент не может купить историю, только начать собирать с нуля.

**Метрика:** количество уникальных видео в базе, глубина истории (дней).

### 2. Ров доверия (Trust Moat)
Публиковать инсайты по нишам (например, "топ болей аудитории фитнес-каналов") — формируется репутация источника истины. Люди начинают ссылаться.

**Метрика:** входящие ссылки, упоминания в профессиональных сообществах.

### 3. Switching Cost
Интеграция с Notion / Airtable / CRM пользователя. Когда инструмент встроен в рабочий процесс команды — переход стоит дорого.

**Метрика:** доля пользователей с активной интеграцией > 30 дней.

### 4. Сетевой эффект (слабый, но реализуемый)
Коллективные теги и разметка болей — если пользователи помечают категории вручную, их разметка улучшает модель для всех.

**Метрика:** % комментариев с пользовательской разметкой.
