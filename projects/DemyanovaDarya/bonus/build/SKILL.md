# Build (Реализация ТЗ)

> **Скилл для дополнительного балла**

Реализует техническое задание по шагам: читает spec.md, планирует порядок реализации, создаёт или редактирует файлы, проверяет что всё работает. Не пишет код наугад — двигается строго по ТЗ.

---

## Triggers

- "реализуй ТЗ", "напиши код по спецификации", "сделай это"
- "implement the spec", "build this feature"
- `/build`

---

## Входные данные

- Файл `spec.md` или текст ТЗ
- Технический стек (если не указан в ТЗ — определить из кода)
- Опционально: конкретный раздел ТЗ для реализации

---

## Алгоритм

### Шаг 1: Прочитать ТЗ

```
Read: spec.md
```

Выписать из ТЗ:
- Список файлов/компонентов к созданию
- API-контракты
- Бизнес-логику
- Edge cases

### Шаг 2: Разведка кодовой базы

```bash
# Определить стек
ls package.json next.config.ts tsconfig.json

# Найти похожие файлы для ориентира
Glob: src/**/*.ts, app/api/**/*.ts
```

### Шаг 3: Составить план реализации

Разбить на атомарные шаги. Пример:

```
1. Создать API-роут /api/feature
2. Написать бизнес-логику в lib/feature.ts
3. Добавить UI-компонент FeatureBlock.tsx
4. Подключить компонент на странице
5. Проверить в браузере
```

### Шаг 4: Реализовать по шагам

Каждый шаг:
- Написать код
- Убедиться что нет синтаксических ошибок
- Не выходить за рамки ТЗ (вне скоупа — не трогать)

### Шаг 5: Запустить и проверить

```bash
npm run dev
# или
npm run build
```

Проверить по критериям готовности из ТЗ.

---

## Правила реализации

- Следовать существующим соглашениям в коде (имена файлов, структура)
- Не добавлять ничего, чего нет в ТЗ
- Обрабатывать edge cases явно — не молча игнорировать
- Переменные окружения — только через `.env.local`, никогда в код
