# Interface Analysis

> **Скилл для дополнительного балла**

Глубокий анализ логики работы продукта через его интерфейсы: пользовательские сценарии, экраны, API, структуры данных. Результат — понимание того как устроен продукт изнутри, без доступа к коду.

---

## Triggers

- "разбери как работает", "проанализируй интерфейс", "изучи логику"
- "analyze how X works", "reverse engineer the UX"
- `/interface-analysis`

---

## Входные данные

- Название продукта / URL сервиса
- Фокус: весь продукт или конкретная функция
- Цель анализа: написать аналог / понять конкурента / написать ТЗ

---

## Алгоритм

### Шаг 1: Сбор поверхностной информации

```
WebSearch: "{продукт} how it works"
WebSearch: "{продукт} API documentation"
WebSearch: "{продукт} onboarding flow"
WebFetch: сайт продукта, страница /features или /how-it-works
```

### Шаг 2: Анализ пользовательских сценариев

Восстановить основные user flows:

```
Регистрация → Онбординг → Ключевое действие → Повторное использование
```

Для каждого экрана описать:
- Что видит пользователь
- Какие действия доступны
- Какие данные требуются
- Что происходит после

### Шаг 3: Анализ API (если публичный)

```
WebFetch: docs.{продукт}.com/api
```

Выписать:
- Основные эндпоинты
- Структуры входных / выходных данных
- Модели аутентификации

### Шаг 4: Выявление неочевидной логики

Найти:
- Как продукт обрабатывает edge cases (пустой экран, ошибка, лимит)
- Какова бизнес-логика за каждым действием
- Где продукт монетизируется / где ограничивает бесплатный план

### Шаг 5: Схема

Составить текстовую схему основных сущностей и связей между ними.

## Вывод пользователю

```
Анализ: Объяснятор (AI-тьютор)
Фокус: загрузка документа → вопрос → ответ
────────────────────────────────────────────
User Flow:
[Загрузить PDF] → [Документ парсится] → [Задать вопрос]
→ [Релевантные чанки извлекаются] → [Ответ с цитатами]

Ключевые экраны:
1. Загрузка: drag-and-drop, прогресс-бар, превью документа
2. Чат: поле ввода, история, подсветка источников

Бизнес-логика:
- Чанкинг при загрузке, хранение в sessionStorage
- Retrieval через keyword scoring (не векторный поиск)
- Каждый вопрос → топ-4 чанка → prompt → ответ

Сущности: Document, Chunk, Message, Session
Ограничения: только текстовые форматы, сессия не сохраняется
```
