# REPORT — Итоги разработки

## Что сделано

Реализован полнофункциональный MVP за один спринт разработки.

### Функционал
- Загрузка документов: PDF, TXT, Markdown, вставка текста
- Чат с тьютором — строго по загруженному документу
- Квиз (5 вопросов с объяснениями)
- Флеш-карточки с CSS-анимацией переворота
- Практические задачи с прогрессивными подсказками
- «Объяснить» при выделении текста
- Сохранение и экспорт заметок в Markdown
- Синхронизация активного раздела в сайдбаре
- Демо-режим без загрузки файла
- Деплой на Vercel: [obyasnyator.vercel.app](https://obyasnyator.vercel.app)

## Ключевые технические решения

**Retrieval без векторной базы данных.** Для MVP выбран keyword scoring — подсчёт совпадений слов из вопроса с текстом каждого чанка. Нулевая инфраструктура, мгновенный старт. Работает хорошо когда ученик спрашивает в терминах документа — а именно так происходит при изучении материала.

**Две модели под разные задачи.** Чат тьютора использует Sonnet (качество важнее цены), генерация квиза и карточек — Haiku (в ~5x дешевле, справляется с batch-генерацией).

**Stateless backend.** Все данные в sessionStorage браузера — никаких баз данных, никаких пользователей. Для учебного инструмента это правильно: конспекты и книги — чувствительные данные.

## Что не вошло в MVP

- Семантический поиск (embeddings) — запланировано в Этапе 2
- Аутентификация и облачное хранение — Этап 3
- Трекинг прогресса обучения — Этап 4

## Выводы

Инструмент закрывает реальную боль: ученик может работать с материалами в любое время без репетитора. Ключевое ограничение MVP — данные не сохраняются между сессиями. Это главный приоритет следующей итерации.
