# Как работать с проектом

Практическое руководство для разработчика.

---

## Запуск на своём компьютере

1. Перейди в папку с проектом: `cd obyasnyator`
2. Установи зависимости: `npm install`
3. Скопируй файл с переменными окружения: `cp .env.local.example .env.local`
4. Открой `.env.local` и вставь свой ключ от Anthropic API
5. Запусти сервер разработки: `npm run dev`
6. Открой браузер на `http://localhost:3000`

---

## Публикация на Vercel

Любой коммит в ветку `main` автоматически запускает новый деплой — ничего дополнительно делать не нужно.

Если нужно передеплоить вручную: зайди на vercel.com, открой проект, вкладка «Deployments» → «Redeploy».

Переменные окружения (в том числе ключ Anthropic) хранятся в разделе «Settings → Environment Variables» в настройках проекта на Vercel.

---

## Добавить новый маршрут к серверному API

1. Создай файл `app/api/название/route.ts`
2. Внутри используй готовый клиент из `lib/ai.ts` — там уже есть клиент Anthropic, функция сборки контекста и парсер ответов
3. Добавь кнопку вызова в `TutorPanel.tsx` или создай новое модальное окно в `components/workspace/`
4. Подключи модальное окно в `app/workspace/page.tsx`

---

## Добавить новый режим работы тьютора

1. Добавь кнопку быстрого действия в `TutorPanel.tsx`
2. Если это продолжение чата — передай подготовленный промпт в функцию `sendMessage()`
3. Если это отдельное окно — добавь обратный вызов `onOpen*` в параметры и открывай окно через `setOpenModal()`

---

## Изменить используемые модели

Открой `lib/ai.ts` и найди константы в начале файла:

```ts
export const TUTOR_MODEL = "claude-sonnet-4-6";       // модель для чата
export const GEN_MODEL = "claude-haiku-4-5-20251001"; // модель для генерации
```

Тьютор (чат) использует более мощную модель. Генерация квиза, карточек и задач — более быструю и дешёвую.

---

## Обновить ключ Anthropic

- Локально: отредактируй `.env.local`, замени значение `ANTHROPIC_API_KEY`
- На продакшене: Vercel → Settings → Environment Variables → найди `ANTHROPIC_API_KEY` → Edit

После замены на локальном сервере подожди пару секунд — в логах терминала появится сообщение `Reload env: .env.local`, это значит новый ключ подхвачен.

---

## Экспорт заметок

В шапке рабочего пространства есть кнопка «Экспорт заметок» — она скачивает файл в формате Markdown. Кнопка активна только если есть хотя бы одна сохранённая заметка.

---

## Тестирование вручную

1. Открой `http://localhost:3000`
2. Нажми «Демо» — откроется рабочее пространство с тестовыми данными без необходимости загружать файл
3. Загрузи настоящий документ в формате PDF, TXT или MD — проверь что текст корректно извлекается
4. Протестируй квиз, карточки и практические задания — модальные окна должны открываться и генерировать содержимое
5. Проверь сохранение заметок и их экспорт
6. Убедись что в консоли браузера нет ошибок

## Структура проекта

```
app/
  page.tsx                — главная страница (лендинг)
  workspace/page.tsx      — рабочее пространство

  api/
    upload/route.ts       — загрузка и обработка документа
    tutor/route.ts        — чат с тьютором
    quiz/route.ts         — генерация квиза
    flashcards/route.ts   — генерация карточек
    practice/route.ts     — генерация практических задач

components/workspace/     — компоненты рабочего пространства
components/ui/            — базовые элементы интерфейса

lib/
  ai.ts                   — клиент Anthropic и вспомогательные функции
  session.ts              — сохранение состояния в браузере
  mock-data.ts            — тестовые данные для демо-режима

types/index.ts            — все типы данных проекта
```
