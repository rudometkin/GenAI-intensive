# PLAN — Развитие продукта

## Этап 1 (сделано): MVP

- Загрузка PDF, TXT, Markdown
- Чат тьютора с keyword-retrieval (top-4 чанка)
- Квиз, флеш-карточки, практические задачи
- Сохранение и экспорт заметок
- Демо-режим без загрузки файла
- Деплой на Vercel

## Этап 2: Улучшение качества ответов

- **Что делаем:** заменить keyword scoring на семантический поиск (embeddings). Сейчас тьютор находит релевантные фрагменты по совпадению слов — это работает когда вопрос сформулирован в терминах документа. Семантический поиск найдёт нужное даже если вопрос задан своими словами
- **Метрика:** доля вопросов, получивших релевантный ответ (оценка пользователя 4+/5)
- **Инструменты:** Supabase pgvector или Pinecone

## Этап 3: Сохранение прогресса

- **Что делаем:** аутентификация + облачное хранение. Сейчас все данные в браузере — закрыл вкладку, потерял прогресс. Добавить вход через аккаунт, библиотеку документов, историю квизов
- **Метрика:** retention на 7-й день (цель: 40%+)
- **Инструменты:** Clerk для auth, Supabase для хранения

## Этап 4: Трекинг обучения

- **Что делаем:** показывать ученику прогресс — какие карточки уже знает, какие темы провалил в квизе, что нужно повторить. Интервальное повторение для карточек (алгоритм Leitner)
- **Метрика:** % учеников, вернувшихся к карточкам через 3+ дня
- **Инструменты:** кастомная логика на основе результатов квизов

## Этап 5: Интеграция в Private.Education

- **Что делаем:** куратор загружает материал → ученик получает ссылку на рабочее пространство. Куратор видит статистику: сколько времени ученик провёл, какие вопросы задавал, результаты квизов
- **Метрика:** % учеников, использующих инструмент между занятиями
