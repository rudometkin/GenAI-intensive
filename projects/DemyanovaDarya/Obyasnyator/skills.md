## Как работает искусственный интеллект

### Системный промпт тьютора должен содержать:

- **Роль:** тьютор отвечает исключительно по материалу загруженного пользователем документа, не придумывает информацию от себя
- **Контекст:** фрагменты документа, наиболее близкие к заданному вопросу
- **Язык:** ответ должен быть на том же языке, что и вопрос пользователя
- **Стиль:** объяснение пошагово, ключевые термины выделены жирным шрифтом

### Какие модели используются:

| Задача | Модель |
|--------|--------|
| Чат тьютора | `claude-sonnet-4-6` |
| Квиз, карточки, практика | `claude-haiku-4-5-20251001` |

Для диалога используется более мощная и дорогая модель — качество ответов здесь критично. Для генерации структурированного содержимого (квиз, карточки) достаточно более быстрой и дешёвой модели.

### Как выбираются нужные фрагменты документа:

Реализован поиск по ключевым словам без использования векторных баз данных:
1. Слова из вопроса пользователя сравниваются с текстом каждого фрагмента документа
2. Считается, сколько раз слова вопроса встречаются в каждом фрагменте
3. Отбираются 4 фрагмента с наибольшим совпадением
4. Их порядок в документе восстанавливается перед отправкой в модель

Реализовано в `lib/ai.ts`, используется в `app/api/tutor/route.ts`.

## Обработка документов

Загрузка и разбивка документа происходит в `app/api/upload/route.ts`.

Поддерживаемые форматы: PDF, TXT, Markdown, а также вставка текста напрямую.

Алгоритм разбивки на фрагменты:
1. Текст делится по двойным переносам строки и заголовкам
2. Если получившийся фрагмент слишком длинный — делится на части по ~800 символов с перекрытием в 50 символов
3. Заголовок фрагмента — первая строка (не более 80 символов)

---

## Получение структурированных ответов от модели

При генерации квиза, карточек и задач модели явно указывается вернуть только массив в формате JSON без каких-либо пояснений. На случай если модель всё равно обернёт ответ в блок с кодом — используется функция `parseJSON()` из `lib/ai.ts`, которая очищает ответ перед разбором.

## Хранение состояния в браузере

- Загруженный документ хранится в хранилище браузера под ключом `__doc__`
- Демо-режим включается установкой значения `__demo__ = "1"` в том же хранилище
- История чата и заметки сохраняются автоматически при каждом изменении
- Вся логика работы с хранилищем сосредоточена в `lib/session.ts`

## Компоненты интерфейса

**Расположение колонок рабочего пространства:**
```
[Боковая панель 240px] | [Просмотр документа — основная область] | [Панель тьютора 360px]
```

**Добавить новое модальное окно:**
1. Создай компонент с использованием базового `<Modal>` из `components/ui/Modal.tsx`
2. Добавь в параметры: `open`, `onClose`, `docName`, `chunks`
3. Добавь состояние в `app/workspace/page.tsx`: переменная для управления видимостью окна
4. Размести компонент рядом с остальными модальными окнами в разметке

**Всплывающие уведомления:**
```ts
const { toast, show: showToast, dismiss } = useToast();
showToast("Текст уведомления");
```

## Что проверить перед публикацией

1. Запусти проверку типов: `npm run type-check` — ошибок быть не должно
2. Проверь демо-режим по адресу `/workspace?demo`
3. Загрузи реальный PDF — убедись что текст извлекается корректно
4. Открой квиз, карточки, практику — все окна должны работать
5. Сохрани заметку и экспортируй её
6. Убедись что в консоли браузера нет ошибок
