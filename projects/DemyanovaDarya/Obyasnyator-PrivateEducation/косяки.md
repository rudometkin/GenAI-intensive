# Известные проблемы

Задокументированные баги, ограничения и подводные камни проекта.

---

## Ошибка баланса при работе с Anthropic

**Проблема:** ключ создан, но при обращении к API возвращается ошибка о недостатке средств.

**Симптом:** сообщение `"Your credit balance is too low to access the Anthropic API"`.

**Причина:** ключ привязан к аккаунту, а не к балансу. Можно создать десять новых ключей — все они вернут ту же ошибку, если на счету аккаунта нет средств.

**Как исправить:** зайди на `console.anthropic.com` → Settings → Billing. Там должна быть конкретная сумма остатка, а не просто привязанная карта.

## Конфликт библиотеки для чтения PDF со средой выполнения

**Проблема:** библиотека для чтения PDF-файлов работает только в среде Node.js и несовместима с режимом Edge (облегчённая серверная среда Vercel).

**Симптом:** ошибка компиляции или сообщение `Module not found` при публикации на Vercel.

**Решение:** в файле маршрута обязательно указывать `export const runtime = "nodejs"`, а в конфигурационном файле `next.config.ts` добавить `serverExternalPackages: ["pdf-parse"]`.

## Некорректные заголовки для коротких документов

**Проблема:** если документ содержит меньше трёх абзацев, алгоритм разбивки создаёт один фрагмент без заголовка.

**Симптом:** в боковой панели вместо заголовка раздела отображаются первые 80 символов текста.

**Статус:** некритично для первой версии продукта — длинные документы (лекции, статьи) обрабатываются корректно.

## Хранилище браузера в режиме инкогнито на iPhone и iPad

**Проблема:** в браузере Safari при работе в режиме инкогнито хранилище данных браузера может быть недоступно.

**Симптом:** пользователь загружает документ, но рабочее пространство оказывается пустым или происходит перенаправление на главную страницу.

**Обходной путь:** обращение к хранилищу обёрнуто в блок обработки ошибок — при недоступности хранилища приложение продолжает работу без сохранения данных.

## Очень длинные документы выходят за пределы контекстного окна

**Проблема:** если документ содержит более 100 тысяч токенов, весь его текст физически не помещается в контекст модели за один запрос.

**Текущее решение:** в запрос передаются только 4 наиболее релевантных фрагмента, отобранных по поиску ключевых слов.

**Ограничение:** если вопрос сформулирован иначе, чем текст в документе, алгоритм может пропустить релевантный фрагмент.

**Планируемое решение:** семантический поиск на основе векторных представлений текста — запланировано для второй версии.

## Сложная разметка в ответах тьютора

**Проблема:** функция отображения разметки в панели тьютора написана вручную и покрывает не все возможные случаи.

**Симптом:** сложная вложенная разметка (таблицы, вложенные списки) может отображаться некорректно.

**Обходной путь:** для первой версии достаточно. При необходимости заменить на готовую библиотеку `react-markdown`.

## Неиспользуемая зависимость в проекте

**Проблема:** в файле зависимостей проекта (`package.json`) осталась библиотека для работы с OpenAI, хотя проект переключился обратно на Anthropic.

**Статус:** на работу приложения не влияет, но увеличивает размер итогового сборки. Можно удалить командой `npm uninstall openai`.

## Кеширование переменных окружения при локальной разработке

**Проблема:** сервер разработки Next.js кеширует файл с переменными окружения. После замены ключа нужно подождать несколько секунд.

**Симптом:** после обновления ключа в `.env.local` продолжает использоваться старый ключ.

**Решение:** дождись сообщения `Reload env: .env.local` в логах терминала — это подтверждение что новый ключ применён.
