# PROJECTCLONE — Клонирование внешнего сервиса

## 1. Выбранный сервис

**Benchling** — [benchling.com](https://benchling.com)

Лидирующая ELN+LIMS-платформа для biotech и фармы. После санкций 2024 г. полностью недоступна в России. Гипотеза: moat Benchling держится не на технологии (React-интерфейс воспроизводим), а на **данных** (накопленные протоколы экспериментов) и **switching cost** (переобучение команды + история данных нельзя легко перенести). Визуальный интерфейс — это фасад, а не ров.

## 2. Необходимые скиллы для сборки

- `Claude Code` — генерация React/TypeScript компонентов по описанию экрана, отладка логики Gantt-диаграммы
- `Cursor` — рефакторинг и адаптация компонентов, работа с типами
- `Supabase` — схема БД, RLS-политики, Edge Functions, Auth

## 5. Логика

**Как создавали:**
1. Определили ключевые экраны Benchling: дашборд, проекты, инвентарь, планировщик
2. Спроектировали схему БД под multi-tenant (organizations → profiles → projects → tasks → reagents)
3. Собрали фронтенд на React + TypeScript + Tailwind по визуальному референсу
4. Подключили Supabase: Auth, RLS-политики, Edge Function для JWT custom claims
5. Добавили мобильную адаптацию и задеплоили на Vercel

**Самое сложное:**
Gantt-диаграмма — синхронизация высот строк между левой колонкой (имена задач) и правой (временная шкала) при динамическом контенте. Решалось 6 итерациями: от фиксированной высоты к `h-16` с `flex-wrap`.

**Открытые вопросы:**
- Как воспроизвести ELN-core Benchling: структурированные нотации, версионирование, электронная подпись протоколов
- Как добавить AI-слой (RAG по протоколам) без деградации производительности

## 6. ⭐ Ров

**Данные:**
Benchling накапливает миллионы протоколов экспериментов за годы. Данные становятся активом — AI-модели, обученные на них, дают инсайты, недоступные конкурентам. Стратегия: с первого дня хранить все действия в audit_log, строить версионирование протоколов — через 2–3 года это станет уникальным датасетом по российской науке.

**Switching cost:**
История экспериментов, привязанная к реагентам и исполнителям, создаёт операционную зависимость. Чем дольше лаборатория работает в системе, тем дороже миграция. Стратегия: глубокая интеграция в ежедневные процессы (учёт расходников, дедлайны, GxP-отчёты) раньше, чем появятся конкуренты.

**Регуляторка:**
GxP/GLP-сертификация создаёт барьер входа: конкурент должен пройти тот же длинный путь валидации. Стратегия: audit_log и электронная подпись протоколов → первая GxP-сертификация → контракты с фармой → репутационный барьер.
