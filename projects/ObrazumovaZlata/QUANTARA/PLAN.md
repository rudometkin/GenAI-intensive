# PLAN — Построение рвов на 3–6 месяцев

## Ров 1: Данные

- **Что делаем:** Накапливаем структурированные данные о протоколах, реагентах и результатах экспериментов внутри платформы. Добавляем версионирование протоколов (ELN-core) и audit_log по всем действиям. Со временем — обезличенные бенчмарки по типам экспериментов.
- **Метрика:** Количество протоколов/записей на организацию; retention через 90 дней (данные накоплены — уйти сложнее).
- **Что мешает:** Медленный онбординг — лаборатории вводят данные руками. Нужны шаблоны протоколов и импорт из Excel.

## Ров 2: Switching cost

- **Что делаем:** Интегрируемся в повседневные процессы лаборатории: Gantt для планирования экспериментов, учёт реагентов с алертами на минимальный остаток, история аудита для GxP-отчётности. Чем глубже интеграция в процессы, тем дороже миграция.
- **Метрика:** DAU/MAU ratio; количество модулей, которые команда использует одновременно (цель — 3+ модуля на org).
- **Что мешает:** Пока нет интеграций с внешними системами (1С, ERP) — крупные орги требуют их до подписания.

## Ров 3: Доверие / Регуляторка

- **Что делаем:** Получаем сертификацию для GxP/GLP-клиентов (фарма, биотех). Audit_log уже заложен в схему БД. Добавляем электронную подпись протоколов и отчёты для регуляторов. Позиционируемся как "отечественное ПО" для получения госсубсидий клиентами.
- **Метрика:** Первый подписанный контракт с фармацевтической компанией или НИИ под GxP-требования.
- **Что мешает:** Сертификация требует времени и юридической экспертизы. На первом этапе — достаточно задокументированного audit trail.
