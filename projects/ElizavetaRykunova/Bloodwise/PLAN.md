# PLAN.md — план построения рвов, 3–6 месяцев

## Методология

Ров — это конкурентное преимущество, которое сложно скопировать. Для Bloodwise выбраны три типа: данные, доверие и switching cost.

---

## Ров 1 — Данные

### Суть
Накапливать обезличенные паттерны: какие показатели чаще отклоняются в связке, какие формулировки расшифровки пользователь читает дольше всего, по каким отклонениям чаще задают уточняющие вопросы.

### Что делаем по месяцам

**Месяц 1–2:**
- Добавить логирование событий (анонимно): какие показатели были введены, были ли отклонения, сколько времени провёл на результате
- Хранить в Supabase (бесплатный tier достаточен для MVP)

**Месяц 3–4:**
- Запустить A/B тест формулировок расшифровки: медицинский стиль vs. разговорный. Смотреть на time-on-page и клики на «задать уточняющий вопрос»
- Начать сбор тегов: какой тип анализа (ОАК / биохимия / гормоны / витамины)

**Месяц 5–6:**
- Выгрузить топ-20 паттернов отклонений → использовать для улучшения промптов
- Написать публичный отчёт «что показывают анализы россиян» — контент-маркетинг + демонстрация данных

### Метрика, что ров появляется
- 10 000+ расшифрованных анализов в базе
- Появляются паттерны, которых нет в открытых справочниках (специфика российской популяции)

### Что мешает
- Нужно тщательно выстроить обезличивание — медицинские данные чувствительны, GDPR / 152-ФЗ
- Пользователи могут не давать согласие на сбор данных

## Ров 2 — Доверие

### Суть
Медицина — это high-stakes область. Доверие строится медленно и не копируется. Верифицированные врачи + прозрачная методология > любой конкурент без этого.

### Что делаем по месяцам

**Месяц 1–2:**
- Найти 2–3 практикующих терапевтов / гематологов для advisory board (достаточно 2–3 часа в месяц на проверку промптов)
- Добавить на сайт: «расшифровка проверена врачами — список консультантов»
- Написать публичную методологию: как работает расшифровка, какие ограничения

**Месяц 3–4:**
- Собрать 50 отзывов реальных пользователей с указанием контекста («сдавала ОАК при беременности, помогло понять зачем пересдавать»)
- Партнёрство с одним медицинским блогером / каналом в Telegram (не реклама — совместный разбор кейса)

**Месяц 5–6:**
- Пресс-упоминание в одном медиа о здоровье (ЗОЖник, Wonderzine Health, МедПортал)
- Добавить «задать вопрос врачу» как paid feature — платный доступ к консультации подтверждает серьёзность сервиса

### Метрика, что ров появляется
- NPS > 50
- % пользователей, рекомендующих сервис по запросу «расшифровка анализов» в поиске
- Упоминания без платного продвижения

### Что мешает
- Врачи не хотят брать на себя ответственность за AI-расшифровки
- Нужен юридически грамотный дисклеймер

## Ров 3 — Switching Cost

### Суть
Как только у пользователя есть история анализов в динамике — уйти означает потерять всё. Чем больше данных загружено, тем выше стоимость переключения.

### Что делаем по месяцам

**Месяц 1–2:**
- Запустить историю анализов: сохранение расшифровок с датой и типом
- Базовый график динамики хотя бы по двум показателям (гемоглобин, лейкоциты)

**Месяц 3–4:**
- Напоминания: «прошло 3 месяца с последнего ОАК — самое время пересдать»
- Автоматическое сравнение нового анализа с предыдущим: «гемоглобин вырос с 112 до 118 — положительная динамика»

**Месяц 5–6:**
- Экспорт истории в PDF (красивый отчёт для врача) — делает данные переносимыми, но заодно показывает ценность накопленного
- Интеграция с одной лабораторией (Инвитро или Гемотест) — автоматическая загрузка без ручного ввода → критически снижает трение и повышает частоту использования

### Метрика, что ров появляется
- Retention через 3 месяца > 25%
- % пользователей с 3+ загруженными анализами > 15%
- Churn rate снижается с каждым дополнительным загруженным анализом (должна быть обратная зависимость)

### Что мешает
- Интеграция с лабораториями требует B2B-переговоров (долго)
- История анализов требует авторизации → трение при онбординге
