# PROJECTCLONE — клонирование Lab4U / Helixa

## Какой сервис выбрала и почему

**Lab4U** — мобильное приложение для расшифровки анализов крови (Россия). Пользователь фотографирует бланк или вводит значения вручную, получает объяснение каждого показателя с цветовой индикацией отклонений.

Выбрала по трём причинам:
1. Прямое пересечение с моим проектом Bloodwise — могу сравнить архитектурные решения
2. Продукт решает реальную боль: люди массово гуглят «что значит гемоглобин 110» после каждого похода в лабораторию
3. Понятный функциональный объём — можно клонировать ядро за разумное время

---

## Продуктовые модули и чем закрываются

| Модуль | Что делает | Чем закрывается |
|---|---|---|
| Input | Текст / фото бланка | HTML форма + FileReader API |
| Parser | Распознавание показателей | Regex по ключевым словам (Hb, лейкоциты, СОЭ...) |
| Reference DB | Нормы по полу/возрасту | JSON-справочник с диапазонами |
| Interpreter | Объяснение отклонений | Claude API (system prompt с ролью медассистента) |
| Output | Визуализация результата | Карточки с цветовой индикацией |
| History | История анализов | localStorage |

---

## Какие скилы нужны были для сборки

- **HTML/CSS/JS** — вёрстка интерфейса, drag-and-drop загрузка файлов
- **FileReader API** — конвертация изображений в base64
- **Regex** — парсинг текста анализов (нестандартные форматы разных лабораторий)
- **Claude API** — мультимодальные запросы (текст + изображение), prompt engineering
- **Prompt engineering** — структурированный медицинский промпт с нужным форматом вывода

## Логика создания

### Порядок сборки
1. Сделала базовую форму ввода (textarea)
2. Добавила regex-парсер для извлечения числовых значений
3. Подключила Claude API с системным промптом
4. Добавила загрузку изображения (base64 → multimodal request)
5. Сделала summary-карточки с цветовой индикацией

### Что было самым сложным
- **Regex для разных форматов лабораторий** — каждая лаборатория форматирует бланк по-своему. «Hb», «Гемоглобин», «HGB», «Hemoglobin» — всё одно и то же, но написано по-разному. Решила через набор альтернатив в regex.
- **Структура промпта** — первые версии давали либо слишком медицинский язык (непонятный пациенту), либо слишком упрощённый (без полезной информации). Итоговый промпт явно задаёт формат: краткое резюме → по каждому показателю → рекомендации.

### Что осталось открытым
- PDF-парсинг: сейчас PDF передаётся как изображение (ненадёжно для многостраничных бланков), нужен pdf.js для извлечения текста
- Нормы зависят от пола и возраста — сейчас используются усреднённые
- API-ключ лежит на клиенте — для продакшена нужен серверный прокси

## ⭐ Как сделать ров (задача со звёздочкой)

### 1. Ров через данные
Собирать обезличенные паттерны: какие показатели чаще отклоняются в связке, какие вопросы задают пользователи после расшифровки. Через 50k+ анализов появляется статистика, которую нельзя воспроизвести без такой базы. Lab4U сделать это не может без пересборки продукта.

### 2. Ров через доверие
Верификация расшифровок практикующими врачами + публичный список консультантов. Пользователь видит «проверено терапевтом Ивановой А.В., 15 лет практики» — это не копируется конкурентом за ночь.

### 3. Ров через switching cost
История анализов в динамике — как только пользователь загрузил 3–4 бланка за полгода, уйти означает потерять всю историю и тренды. Усиливается напоминаниями: «сдайте повторный ОАК через 3 месяца, чтобы отследить динамику».

### 4. Ров через сеть
Интеграция с лабораториями (Инвитро, Гемотест, CMD) — автоматическая загрузка результатов из личного кабинета лаборатории. Переговоры с лабораториями требуют времени, но создают B2B-канал, который стартап за неделю не повторит.
