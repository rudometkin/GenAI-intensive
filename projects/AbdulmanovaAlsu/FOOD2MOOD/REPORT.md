# REPORT — Что изменилось за две недели

## Контекст

FOOD2MOOD — AI-ассистент для онлайн-меню ресторанов. Встраивается в QR-меню как плагин: помогает гостю выбрать блюда под его предпочтения, ограничения и уровень голода.

Текущая версия — демонстрационный прототип для потенциальных партнёров (сервисы QR-меню).

---

## Что сделали

### Архитектура прототипа
Собрали работающий фронтенд-прототип на React + TypeScript + Vite. Структура:

```
src/
├── app/
│   ├── pages/          — Onboarding, Quiz, Menu, Loading
│   ├── components/     — ChatBot, MenuItem, MenuDetail, Basket
│   ├── context/        — PreferencesContext, BasketContext
│   └── utils/          — personalization.ts, recommendations.ts
└── styles/
```

**Ключевые решения:**
- Вся логика рекомендаций живёт в `recommendations.ts` — rule-based алгоритм сопоставления предпочтений гостя с атрибутами блюд
- Профиль пользователя хранится в `localStorage` через `PreferencesContext`
- Чат реализован как state machine внутри `ChatBot.tsx` — каждый шаг (лояльность → сессионные вопросы → профиль → рекомендации) — отдельное состояние

### Что работает в прототипе
- Онбординг (показывается один раз)
- Квиз: уровень голода, что хочется, напитки
- Персональные рекомендации из меню на основе ответов
- Карточка блюда с персонализированным описанием
- Корзина (локально)
- Сохранение профиля в localStorage

## Что переделали

- Отказались от Whisper/аудио-транскрипции для персонализации — слишком тяжело для прототипа, заменили на статичный квиз
- OTP-авторизация оставлена как заглушка — реализовывать SMS на этапе демо не имеет смысла
- Корзина отвязана от реального API партнёра — работает локально для демонстрации UX
