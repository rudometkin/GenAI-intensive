# REPORT -- Что изменилось за две недели

Отчет на уровне архитектуры. Период: 10--20 апреля 2026.

## Решения

### 1. ISCP Proxy per user

Каждому пользователю -- отдельный резидентный прокси (2500 руб/мес).

Профи.ру банит аккаунты при мульти-аккаунтном доступе с одного IP. Потеря аккаунта -- катастрофа: отзывы, рейтинг, история откликов. Дешевле платить за прокси, чем восстанавливать аккаунт.

### 2. Разделение CustDev на проблемное и решенческое интервью

- Гайд A -- только проблема (12--15 вопросов). Проводится первым.
- Гайд B -- решение. Проводится ПОСЛЕ подтверждения боли на 10--15 интервью по гайду A.

Mom Test требует разделения: решенческие вопросы на раннем этапе дают социально желательные ответы. Респондент хвалит идею из вежливости, а не из потребности.

### 3. CDDC v3.1 заменяет exit criteria

Customer Development Design Canvas (5 Discovery + 4 Validation + итоговая гипотеза) вместо плоского GO/NO-GO списка.

Методология BTG-4 -- декомпозиция "от денег назад". Плоский чеклист не показывает, на каком этапе гипотеза ломается. CDDC делает это явным.

### 4. Гибридная монетизация

Было: фикс 7500 руб/мес.
Стало: 990 руб/мес фикс + 2500 руб за подтвержденного ученика.

Оба респондента (R1, R2) хотят платить за результат, не за процесс. Фикс 7500 вызывает сопротивление -- "а если учеников не будет?". Гибрид снижает порог входа и выравнивает интересы.

## Сделано

- 2 CustDev-интервью проведены (R1 Вова Лопатин, R2 Рома)
- CDDC v3.1 создан: 5 этапов Discovery + 4 этапа Validation
- 15 файлов CustDev-инфраструктуры: гайд A, гайд B, скрининг, рекрутинг, шаблон дебрифа, таблица кодирования
- Реструктуризация файлов проекта: из плоской структуры в interviews/synthesis/old
- Бизнес-план v2 + приложения по фидбеку ментора
- 800+ профилей репетиторов собраны скрейпингом Профи.ру

Итерация: бизнес-план v1 -> фидбек ментора -> v2 -> CustDev R1/R2 -> CDDC v3.1 + смена монетизации.

## Переделано

- **Монетизация:** фикс 7500 -> гибрид 990 + 2500/ученик. Причина -- фидбек с интервью R1 и R2.
- **Exit criteria -> CDDC v3.1.** Методологический апгрейд: от плоского списка к структурированному канвасу с этапами.
- **Структура файлов:** flat -> interviews/{rNN_name}/ + synthesis/ + old/. Причина -- при 15+ файлах плоская структура стала нечитаемой.
