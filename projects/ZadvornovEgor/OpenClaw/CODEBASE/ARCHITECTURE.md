# ПрофиАссистент (OpenClaw) -- Архитектура

> Код является приватным и не публикуется в открытом доступе.
> Доступ к исходному коду по запросу: **@egor_zadvornov** (Telegram)

---

## Архитектура системы

```
[Телефон репетитора]
  |
  | NotificationListenerService
  | -> перехват уведомления Профи.ру
  | -> парсинг заказа (предмет, класс, формат, бюджет)
  |
  | HTTP POST /generate-response
  v
[Наш сервер (FastAPI)]
  |
  | -> Claude API (генерация персонализированного ответа)
  | -> Telegram Bot API (уведомление репетитору)
  |
  | HTTP Response (текст ответа)
  v
[Телефон репетитора]
  |
  | Accessibility Service
  | -> открытие страницы заказа в приложении Профи.ру
  | -> заполнение формы ответа
  | -> отправка отклика
```

## Два варианта архитектуры

### Вариант 1: Server (MVP)

Headless-браузер (Playwright) на VPS, работающий через ISCP резидентный прокси. Сервер самостоятельно заходит на Профи.ру, мониторит заказы и отправляет отклики.

| Параметр | Значение |
|----------|----------|
| COGS на клиента | ~5 000 руб/мес (прокси 2 500 + LLM 1 500 + сервер 1 000) |
| Риск бана | **ВЫСОКИЙ** -- серверный IP, паттерны headless-браузера |
| Маржа при цене 7 500 | ~33% |
| Время запуска | 1-2 недели |

**Плюсы:** быстрый запуск, не требует установки на телефон клиента.
**Минусы:** высокий COGS, риск блокировки аккаунта, сложная поддержка при изменениях на сайте.

### Вариант 2: Android (целевой)

Нативное Android-приложение на телефоне пользователя. Перехватывает push-уведомления от Профи.ру через `NotificationListenerService`, отправляет данные заказа на наш сервер, получает сгенерированный ответ и заполняет форму через `AccessibilityService`.

| Параметр | Значение |
|----------|----------|
| COGS на клиента | ~200 руб/мес (только LLM API) |
| Риск бана | **НИЗКИЙ** -- реальное устройство, реальный IP, человеческие паттерны |
| Маржа при цене 7 500 | ~97% |
| Время запуска | 4-6 недель |

**Плюсы:** минимальный COGS, низкий риск бана, масштабируемость.
**Минусы:** требует установки APK, зависимость от Android-версии клиента, сложнее в разработке.

## Ключевые технические решения

1. **NotificationListenerService** -- системный сервис Android, не требует root. Перехватывает все push-уведомления и извлекает данные заказа.

2. **AccessibilityService** -- позволяет программно взаимодействовать с UI других приложений. Используется для автоматического заполнения формы отклика в приложении Профи.ру.

3. **Claude API через наш сервер** -- клиентское приложение не хранит API-ключи. Все запросы к LLM проксируются через FastAPI-сервер, что позволяет логировать, кэшировать и контролировать расходы.

4. **Telegram-бот** -- уведомляет репетитора о новых заказах и отправленных откликах. Позволяет подтвердить или отменить автоотклик до отправки (опционально).
