# PLAN -- Построение рвов на 3-6 месяцев

Выбранные типы рвов: данные / доверие / switching cost.

## Ров 1: Данные

- **Что делаем:** Копим данные о конверсии ответов (какие формулировки приводят к ученику), паттерны банов (за что банят, через сколько), 800+ профилей репетиторов, воронку по предметам/ставкам/времени суток.
- **Метрика:** Конверсия первого ответа в подтвержденного ученика (baseline: 10-15%, цель: 25%+ через 6 мес)
- **Что мешает:** На 2-5 пользователях данных мало. Конкурент с 100 пользователями за 3 месяца соберет столько же.

## Ров 2: Доверие

- **Что делаем:** Региональные кластеры (МФТИ -> МГУ -> ВШЭ). Реферальная программа (привел 4+ друзей = бесплатно навсегда). Персональный онбординг. TG-канал с контентом для репетиторов.
- **Метрика:** NPS > 40, K-фактор > 0.3 (каждый пользователь приводит 0.3 новых)
- **Что мешает:** R2 прямо сказал: "проблема для узкого круга ребят". Вирусность может быть ограничена. МФТИ -- может быть пузырь.

## Ров 3: Switching cost

- **Что делаем:** AI обучается на стиле конкретного репетитора (его шаблоны, тон, предмет). Система запоминает историю учеников, настройки фильтров, предпочтения. Уйти к конкуренту = начать обучение с нуля.
- **Метрика:** Churn после 3+ месяцев использования < 10% (vs 30% в первый месяц)
- **Что мешает:** На MVP switching cost минимальный -- мало персонализации. Работает только при 3+ месяцах использования.

## Что НЕ является рвом (честно)

- Head start (2-3 месяца не защищают)
- Технология (Claude API + скрейпер -- не rocket science)
- Бренд (при 2-5 пользователях бренда нет)
- Патенты (нет и не будет)

## Timeline

- Месяц 1-2: Server MVP, 2-5 пользователей МФТИ, 20 CustDev-интервью
- Месяц 2-3: PoC Android, пилот на 3-5 пользователях
- Месяц 3-4: 20 пользователей МФТИ, Android как основной
- Месяц 4-6: МГУ/ВШЭ, 50+ пользователей, data flywheel
