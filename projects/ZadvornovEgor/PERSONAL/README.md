# Personal AI Operating System

> Персональная операционная система на базе Claude Code: 164 скилла, 40+ агентов, 80+ команд.

## Мои скилы

10 ключевых скиллов из 164, выбранных по принципу "именно эти скиллы создали артефакты в этой домашке". Полный каталог -- ниже в секции 2.

### 1. Архитектура сис��емы

Система состоит из трёх слоёв, объединённых маршрутной таблицей `routing.md`. Любой запрос пользователя автоматически направляется к нужному инструменту.

**Принцип работы:**

```
Запрос пользователя --> routing.md (маршрутизация) --> Skill/Agent --> Артефакт --> Quality Gate
```

**Компоненты:**

- **Routing table** -- автоматически направляет задачу к нужному скиллу по ключевым словам. Содержит 60+ маршрутов: от генерации изображений до продуктового анализа. Пользователь пишет "давай подумаем" -- система вызывает `/think-through`. Пишет "ресёрч рынка" -- вызывает `/market-research`.

- **Skills (164 шт.)** -- специализированные навыки в формате `dir/SKILL.md` с YAML frontmatter, триггерами и пошаговыми инструкциями. Каждый скилл -- это изолированный модуль с чётким входом и выходом. Скиллы могут вызывать друг друга, образуя цепочки (см. раздел 3).

- **Agents (40+ шт.)** -- автономные субагенты для сложных задач: `code-reviewer` (ревью кода), `bug-hunter` (поиск багов), `security-scanner` (аудит безопасности), `software-architect` (проектирование архитектуры), `test-writer` (написание тестов). Агенты запускаются через систему делегирования с уровнями сложности (Level 0-3).

- **Quality Gates** -- обязательные проверки после изменений: `type-check`, `build`, `lint`, `test`. Систематический дебаг через 4 фазы: Root Cause Investigation, Pattern Analysis, Hypothesis Testing, Implementation. Никаких фиксов без установленной причины.

## 3. Композиция скиллов

Скиллы не работают изолированно -- они вызывают друг друга, образуя цепочки. Пример для полного продуктового анализа:

```
/product-pipeline
  |-- /market-research (исследование рынка)
  |     '-- /ai-search-free (deep web research через Perplexity)
  |-- /jtbd-analysis (сегменты и граф работ)
  |-- /tsa-progon (валидация через 30 AI-аватаров)
  '-- /cddc-canvas (Customer Development Design Canvas)
        '-- /think-through (анализ решений на каждом этапе)
```

Другие цепочки:

```
/skill-creator --> security-scanner agent --> /code-reviewer
/bug-hunter agent --> /systematic-debugging --> /bug-fixer agent
/tg-post --> /de-ai-ify --> tg_client.py (отправка)
```

Принцип: каждый скилл решает одну задачу. Сложные задачи решаются композицией простых скиллов.

## Задачи, под которые планирую найти/собрать скилы

Пять направлений развития системы:

1. **Interview prep** -- автоматическая подготовка к интервью: парсинг вакансии, матчинг с CV, генерация STAR-ответов под конкретные требования, симуляция собеседования через AI-аватар интервьюера.

2. **Weekly digest** -- еженедельная сводка из всех каналов: непрочитанные TG (приоритезация по важности), новые письма Gmail, события календаря, прогресс по задачам Trello. Один файл вместо 5 источников.

3. **Meeting-to-actions** -- из транскрипта встречи (tl;dv, Deepgram) в структурированный список: решения, задачи с ответственными, дедлайны, открытые вопросы. Автоматическая отправка в Trello.

4. **Portfolio case generator** -- из проектного опыта (projects-hub, git history, meeting transcripts) в оформленный кейс: проблема, решение, метрики, визуализация. Формат для Notion-портфолио и PDF.

5. **LinkedIn outreach** -- персонализированные сообщения на основе профиля собеседника: парсинг LinkedIn через Apify, выявление общих интересов, генерация opener через `/de-ai-ify` (без AI-маркеров), A/B тестирование подходов.
