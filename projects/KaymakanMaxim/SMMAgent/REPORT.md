# REPORT — Что изменилось за 2 недели

## Исследование (Customer Discovery)

Провели 8 глубинных интервью:
- 4 SMM-специалиста/руководителя в агентствах (Mosaic, креативные агентства, PR-агентства)
- 4 предпринимателя (бьюти, B2B-стартап в аквакультуре, эксперт-ментор, консалтинг)

**Ключевые инсайты:**
- Главная общая боль агентств — **аналитика**: нет инструмента, который агрегирует LiveDune + TG-STAT + ЯМ + ВК в один отчёт
- Главная боль малого бизнеса — **потеря контроля качества** без микроменеджмента
- Все 8 готовы тестировать бесплатную бета-версию
- Визуальный контент не автоматизируют из-за риска авторских прав (реальный судебный иск в одном из агентств)

## Архитектурные решения

**Принято:**
- Старт с двух сегментов параллельно (агентства + малый бизнес) — разные интерфейсы, одно ядро
- MVP = контент-план + текст постов + автопостинг в Telegram (без визуала на старте)
- Визуал — опциональный модуль с явным предупреждением об авторских правах

**Отложено:**
- Интеграция с LiveDune/TG-STAT API — нужно уточнить доступность API и стоимость
- Модуль интерпретации правок — сложнее технически, откладываем на v2

## Технический прогресс

- Собран Next.js фронтенд для отображения контент-плана (PROJECTCLONE — основа для UI-паттернов)
- Подключён Claude API для генерации текстов
- Реализована базовая генерация изображений через fal.ai и kie.ai
- Написан скрипт парсинга Telegram-каналов через Telethon (мониторинг конкурентов)
- Проведены первые тесты генерации 30-постового контент-плана — результат приемлемый, нужна доработка бренд-голоса
