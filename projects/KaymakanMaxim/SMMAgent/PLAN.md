# PLAN — Построение рвов на 3–6 месяцев

## Ров 1: Данные (бренд-голос + история контента)

- **Что делаем:** при онбординге собираем голос бренда, историю постов, реакции аудитории. Каждый следующий пост обучает модель — что зашло, что нет. Персональный профиль бренда накапливается у нас, не у конкурентов
- **Метрика:** % постов, принятых без правок клиента (цель: 60%+ через 3 мес.) / точность попадания в бренд-голос (оценка клиента 1–5)
- **Что мешает:** нужна история постов и метрик от клиента; не все готовы делиться доступами к аналитике сразу

## Ров 2: Switching cost (интеграции + онбординг)

- **Что делаем:** глубокая интеграция с рабочим процессом клиента — Google Sheets для контент-плана, PostMyPost/Telegram для согласования, LiveDune/TG-STAT для аналитики. Чем дольше клиент работает, тем больше данных внутри системы — уходить дорого
- **Метрика:** время от регистрации до первого опубликованного поста (цель: < 2 часов) / NPS через 30 дней
- **Что мешает:** разные клиенты используют разные инструменты; нужны коннекторы к каждому

## Ров 3: Доверие (кейсы + беты)

- **Что делаем:** 10–15 бета-пользователей из интервью получают бесплатный доступ → собираем кейсы с цифрами (было/стало по времени, охватам, количеству итераций правок). Агентства доверяют агентствам — рефералы внутри рынка
- **Метрика:** 3 публичных кейса с цифрами через 2 мес. / 5+ рефералов через 3 мес.
- **Что мешает:** клиенты не хотят светить данными конкурентам; нужны анонимизированные кейсы или согласие на публикацию
