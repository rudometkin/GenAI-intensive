## 1. Архитектурное мышление

### Ограничение области ответственности (Scope Control)
Одним из ключевых проектировочных решений явилось намеренное сужение функциональности до единственного core use case: `URL → clean output`. Отказ от реализации полного crawl-движка, агентных workflows и enterprise-инфраструктуры позволил сфокусироваться на качестве извлечения и пользовательском опыте вместо масштабируемости системы.

### Разделение ответственности (Separation of Concerns)
Пайплайн обработки разбит на независимые слои: валидация → получение HTML → извлечение контента → трансформация → суммаризация. Каждый слой реализован как самостоятельная функция в отдельном модуле `lib/`, что обеспечивает тестируемость и замену отдельных компонентов без затрагивания остальных.

## 2. Frontend-разработка

### Next.js 16 (App Router)
- Разграничение Server Components и Client Components (`'use client'`)
- Route Handlers (`app/api/scrape/route.ts`) с типизированными запросами и ответами
- Работа с метаданными (`Metadata` API в `layout.tsx`)
- Конфигурация `serverExternalPackages` для исключения нативных Node.js-модулей из бандла

### React 19
- Управление состоянием через `useState`, `useEffect`, `useCallback`
- Мемоизация вычислений через `useMemo` (генерация промпта клонирования)
- Паттерн controlled inputs с `useRef` для управления фокусом

### TypeScript (строгий режим)
- Проектирование интерфейсов (`ScrapeResult`, `ScrapeError`, `HistoryEntry`, `Link`, `Heading`)
- Дискриминированные union-типы для обработки success/error ответов API
- Удовлетворение условий `satisfies` при формировании JSON-ответов

### Tailwind CSS v4
- Построение тёмной цветовой схемы через семантические переменные (zinc-950 → zinc-900 → zinc-800)
- Адаптивная вёрстка с breakpoints `sm:`
- Кастомные scrollbar-стили через `::-webkit-scrollbar`
- Работа с opacity-модификаторами (`ring-indigo-500/30`, `bg-amber-500/10`)

## 3. Backend и обработка данных

### HTML-парсинг (Cheerio)
- Traversal DOM-дерева с использованием CSS-селекторов
- Эвристический алгоритм определения основного контента (density-based scoring)
- Каскадная система удаления шума: семантические тэги → ARIA-роли → CSS-классы
- Нормализация относительных ссылок в абсолютные с учётом origin страницы

### Конвертация разметки (Turndown)
- Настройка правил генерации (`headingStyle: 'atx'`, `codeBlockStyle: 'fenced'`)
- Написание кастомных правил (`addRule`) для пропуска изображений и обработки `<br>`
- Пост-процессинг markdown через регулярные выражения (очистка пустых ссылок)

### Экстрактивная суммаризация (NLP без LLM)
- Реализация алгоритма TF-подобного взвешивания предложений на основе частотности слов
- Применение стоп-слов (stop-words) для нормализации скоринга
- Позиционный буст (position boost) для первых предложений документа
- Фильтрация библиографического шума (DOI, ISBN, PMID, паттерны цитирований)
- Корректное разбиение на предложения при отсутствии пробела между ними

## 4. Работа с опциональными зависимостями

### Bundler-bypass pattern (`eval('require')`)
Освоена техника обхода статического анализа модульной системы Turbopack при работе с опциональными нативными зависимостями. `eval('require')('playwright')` выполняется исключительно в Node.js-окружении и не разрешается бандлером на этапе сборки.

### Graceful degradation
Проектирование системы с двухуровневым fallback: статический HTTP-fetch → Playwright → корректное сообщение об ошибке пользователю. Отсутствие Playwright не нарушает функциональность для подавляющего большинства страниц.
