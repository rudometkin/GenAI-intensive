# UI/UX Правила: Копирка

## Общие принципы
- Полированный, быстрый, минималистичный интерфейс
- Ощущение настоящего продукта, не учебного демо
- Один primary action на весь экран — нет ничего лишнего

---

## Компоненты интерфейса

### 1. URL Input
- Большое поле для вставки URL, занимает центр страницы
- Кнопка **Convert** рядом с полем — основной CTA
- Валидация URL на клиенте перед отправкой
- Placeholder: `https://example.com/article`

### 2. Вкладки результата
После обработки появляется блок с 4 вкладками:

| Вкладка | Содержимое |
|---------|-----------|
| **Markdown** | Чистый markdown: заголовки, абзацы, списки, таблицы, ссылки. Рендерится как preformatted text или с подсветкой |
| **JSON** | Структурированный JSON с полями url, title, content, headings, links. Code block с подсветкой синтаксиса |
| **Summary** | 5–7 ключевых тезисов + 1 абзац summary + "main takeaways". Генерируется через LLM |
| **Links** | Список всех найденных ссылок на странице. Кликабельные, с текстом и href |

### 3. Copy / Download
Кнопки присутствуют на каждой вкладке:
- **Copy Markdown** — копировать markdown в буфер
- **Copy JSON** — копировать JSON в буфер
- **Download .md** — скачать файл
- **Download .json** — скачать файл

### 4. История запросов
- Список последних 5–10 URL
- Показывается под input или в сайдбаре
- Клик по URL повторяет запрос
- Хранится в localStorage (MVP) или в БД по session ID

---

## Состояния интерфейса

| Состояние | Поведение |
|-----------|-----------|
| **Idle** | Пустое поле, CTA активна, история снизу |
| **Loading** | Spinner / progress bar, кнопка Convert заблокирована, текст "Fetching page..." |
| **Success** | Показаны вкладки с результатом, Copy/Download кнопки активны |
| **Error** | Видимое сообщение об ошибке под полем (не ломает интерфейс) |

### Типы ошибок для отображения
- Невалидный URL
- Сайт недоступен / таймаут
- Пустая экстракция (контент не найден)
- JS-рендеринг упал

---

## Визуальный стиль
- Минималистичный дизайн (тёмная или светлая тема)
- Чёткая моноширинная типографика для markdown и code блоков
- Никаких лишних элементов, баннеров, попапов на главной странице
- Состояния loading и error выражены ясно, без агрессивных цветов
