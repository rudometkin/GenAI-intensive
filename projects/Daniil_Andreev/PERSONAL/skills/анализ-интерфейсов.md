# **Скилл «Анализ интерфейсов»**

<u>Назначение</u>

Беру ссылку на сервис или серию скриншотов — и раскладываю чужое решение на продуктовые слои: навигация, ключевые сценарии, точки монетизации, места удержания. Выход используется как сырьё для *клонирования*, *конкурентного анализа* или *постановки ТЗ*.

<u>Когда триггерится</u>

- прямая команда «разложи интерфейс», «разбери продукт», «посмотри, как устроен X»;
- начало задачи по клонированию: нужно понять, *из чего состоит* прототип;
- подготовка к обсуждению с командой: дизайнером, продактом, разработчиком.

<u>Что делает</u>

1. **Карта экранов.** Строю дерево навигации: от лендинга до admin-панели, с указанием того, какие действия скрыты за авторизацией.
2. **Сценарии.** Выделяю *три-пять ключевых путей пользователя*: онбординг, основное действие, расширенное действие, возврат, офбординг.
3. **Точки решения.** Фиксирую места, где интерфейс заставляет пользователя выбирать: тариф, настройка, отказ от пробного периода. Это кандидаты в метрики.
4. **Скрытые фичи.** Отмечаю то, что видно только при определённых условиях: пустое состояние, edge-кейсы, поведение при ошибке сети.
5. **Гипотезы о метриках.** Предполагаю, *что именно измеряет команда продукта* и какие AB-тесты могли бы объяснять текущий вид экранов.

<u>Какой контекст подкладывает</u>

- шаблоны типовых продуктовых сценариев (SaaS, маркетплейс, контент-сервис);
- библиотеку UX-паттернов: empty state, progressive disclosure, паттерны пейволла;
- чек-лист *анти-паттернов* — мест, которые интересно разобрать как то, *что делать не стоит*.

<u>Формат выхода</u>

Отчёт в Markdown:
- **Карта экранов** — маркированный список с отступами;
- **Сценарии** — нумерованный список с пошаговой разбивкой;
- **Точки решения и метрики** — таблица «экран → решение → предполагаемая метрика»;
- **Идеи для своего продукта** — короткий список переиспользуемых приёмов.

<u>Ограничения</u>

Скилл работает со *статическими артефактами* — скриншотами, видеозаписями, описаниями. Серверная логика, закрытая за API, остаётся предметом отдельного исследования.
