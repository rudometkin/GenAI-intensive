# **Скилл «Поиск аналогов»**

<u>Назначение</u>

Превращаю короткую формулировку запроса — например, *«сервис для расшифровки смысла песен»* или *«инструмент для сокращения текста по инфостилю»* — в развёрнутую карту рынка: минимум двадцать конкурентов или аналогов, сравнительная таблица и обоснованная рекомендация, *какой из них интересен для клонирования*.

<u>Когда триггерится</u>

- команда «найди аналоги», «сравни с рынком», «что уже есть похожего»;
- первый шаг любой задачи по клонированию внешнего сервиса;
- подготовка к защите продуктовой гипотезы, где требуется показать насыщенность ниши.

<u>Что делает</u>

1. **Поисковая волна.** Ищу по связкам ключевых слов, Product Hunt, Y Combinator batches, GitHub trending, подборкам в профильных изданиях.
2. **Первичная фильтрация.** Из собранного списка оставляю *20–30 живых* продуктов: стартовавших за последний год либо заметно обновлённых.
3. **Сводная таблица.** Столбцы — функциональность, модель монетизации, стек, уникальное преимущество, *слабое место*. Строки — сервисы.
4. **Топ-3 для клонирования.** Выделяю три, исходя из баланса «интересный UX / посильный технический стек / невысокая зрелость рва».
5. **Рекомендация.** Выбираю один сервис с краткой аргументацией и указанием того, *что именно из него стоит взять*, а что — не стоит.

<u>Какой контекст подкладывает</u>

- шаблон сравнительной таблицы с фиксированным набором столбцов;
- список источников для поисковой волны с их характерными смещениями: Product Hunt перегружает свежими запусками, Reddit — нишевыми инди-проектами;
- критерии *«живого» продукта*: активные релизы, отзывы, присутствие команды в соцсетях.

<u>Формат выхода</u>

- **Таблица** — сводное сравнение всех найденных сервисов.
- **Короткие карточки топ-3** — по одному абзацу на каждый с фокусом на *чему у него учиться*.
- **Рекомендация и риски** — две-три строки о выбранном финалисте и потенциальных ловушках.

<u>Ограничения</u>

Скилл опирается на публично доступные данные. Цифры по выручке и количеству пользователей берутся как оценочные, без претензии на верификацию.
