# weekly

**Триггер:** «итоги недели» / воскресенье / «что сделал за неделю» / «планы на следующую»

## Что делает

1. Читает `tracker.md` — текущий статус по всем компонентам
2. Читает `implementation_plan.md` — что планировалось на эту неделю
3. Просит показать `git log --oneline -20` (или получает сам) — что реально сделано
4. Сравнивает план vs факт
5. Предлагает скорректированный приоритет на следующую неделю
6. Обновляет `tracker.md` если нужно

## Контекст

```
Читать перед ответом:
- kbzhuy/docs/tracker.md
- kbzhuy/docs/implementation_plan.md
- git log --oneline -20 (запустить в kbzhuy/)
```

## Выход

```
## Неделя [N] — итоги

**Сделано:** ...
**Не сделано (и почему):** ...
**Разблокировано:** ...

## Неделя [N+1] — приоритет

1. [задача] — [почему сейчас]
2. ...

## Нужно решить

- [открытый вопрос / блокер]
```
