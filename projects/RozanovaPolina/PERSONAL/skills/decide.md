# decide

**Триггер:** «как лучше сделать X» / «выбрать между A и B» / «нужно принять решение»

## Что делает

1. Читает `decisions/log.md` — нет ли уже похожего решения
2. Читает `decisions/README.md` — формат фиксации
3. Структурирует выбор: варианты → критерии → вывод
4. Фиксирует принятое решение в лог в правильном формате

## Контекст

```
Читать перед ответом:
- kbzhuy/docs/decisions/log.md
- kbzhuy/docs/decisions/README.md
```

## Формат записи в лог

```markdown
## [ДАТА] [Заголовок решения]

**Контекст:** почему встал вопрос
**Варианты:** A / B / C
**Выбрали:** X
**Почему:** критерии выбора
**Последствия:** на что повлияет
```

## Принцип

Минимальное решение, которое решает сегодняшнюю задачу. Не проектировать впрок.
