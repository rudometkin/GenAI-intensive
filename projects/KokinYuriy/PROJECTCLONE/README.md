# PROJECTCLONE — Клонирование doomlingo.ai

## 1. Выбранный сервис

**[Doomlingo](https://www.doomlingo.ai/)** — мобильное приложение, которое превращает думскроллинг в изучение иностранных языков через short-form видео, подобранное по уровню пользователя.

**Почему интересен:** сервис встраивает обучение в уже существующую привычку — листать ленту. Не нужно менять поведение, достаточно заменить контент. С точки зрения moat интересен данными: чем дольше пользователь в приложении, тем точнее рекомендации — петля усиления без явных затрат на удержание. А также можно реализовать следующй функционал: пользователь изучает английский, добавляя неизвестные слова в словарь, а потом в нашем приложении изучает эти слова, как в Anki cards.

**Гипотеза:** русскоязычные пользователи охотнее учат английский, если контент — реальные видео на русском с синхронными субтитрами на двух языках, а не учебные упражнения.

**Клон:** [Вникай](vniklay-main/) — веб-версия для мобильных устройств. 

## 2. Необходимые скиллы для сборки

- `frontend-design` — UI видеоплеера в стиле TikTok/Reels: iPhone-фрейм, свайп-навигация, glassmorphic-контролы, двуязычные субтитры поверх видео
- `feature-dev` — бизнес-логика: синхронизация субтитров по таймкодам, мульти-инпут навигация (свайп + колесо мыши), управление состоянием воспроизведения
- `superpowers:brainstorming` — проработка UX-решений перед реализацией каждого модуля
- `superpowers:test-driven-development` — покрытие логики синхронизации субтитров и навигации

## 5. Логика

**Как создавали:**
1. Изучили UX doomlingo.ai — свайп-навигация, уровень сложности, геймификация
2. Адаптировали идею под русскую аудиторию: реальные интервью вместо учебного контента, двуязычные субтитры вместо упражнений
3. Собрали iPhone-фрейм с immersive UI через `frontend-design` скил
4. Реализовали систему синхронизации субтитров по таймкодам — самая сложная часть
5. Добавили навигацию (касание + прокрут)

**Что было сложнее всего:**
- **Синхронизация субтитров:** real-time сравнение `currentTime` видео с диапазонами `start/end` каждой строки, плюс плавные fade-переходы без лишних ре-рендеров. Решили через `useRef` для мемоизации предыдущего субтитра и 150ms CSS-анимацию.
- **Навигация:** одновременно touch-свайп и wheel-события с разными порогами и cooldown-периодами, чтобы не было случайных переключений сцен.

**Вопросы, которые остались:**
- Как масштабировать контент без ручной разметки каждого видео
- Как добавить уровни сложности — в текущей версии все видео равнозначны по сложности

## 6. ⭐ Ров (задача со звёздочкой)

**Ров 1 — Данные**
Каждое взаимодействие пользователя (пауза на субтитре, повтор сцены, свайп влево) — сигнал о понимании. Накапливая эти данные, сервис строит поведенческий профиль уровня английского, который невозможно перенести в другое приложение. Метрика: точность предсказания «слишком сложно / слишком просто» через 30 дней использования.

**Ров 2 — Switching cost**
Пользователь накапливает историю просмотров, выученные слова из субтитров и streak. Уход = потеря прогресса. Чем дольше в приложении — тем выше стоимость переключения. 

**Ров 3 — Контент (библиотека)**
Кураторская библиотека видео с разметкой таймкодов субтитров — трудоёмкий актив. Автоматизация через Whisper/ASR снижает стоимость добавления контента, но качество ручной разметки остаётся барьером входа. Новый конкурент не соберёт за месяц 10 000 размеченных видео.
