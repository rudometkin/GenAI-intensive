# BONUS — Набор скилов для написания ТЗ

Набор из 3 пошаговых скилов, покрывающих полный цикл создания технического задания: от сбора требований до проверки готового документа.

## Почему ТЗ

Написание ТЗ — это процесс, который повторяется в каждом проекте и каждый раз требует одних и тех же шагов: понять контекст, структурировать требования, убедиться что ничего не пропущено. Это идеальная задача для автоматизации через набор скилов — каждый скил отвечает за один этап, вместе они образуют пайплайн.

## Скилы

| Скил | Когда | Что делает |
|------|-------|-----------|
| [TZ_Context_Gatherer](tz-context-gatherer.md) | Перед написанием ТЗ | Задаёт 5 блоков вопросов, собирает резюме контекста |
| [TZ_Writer](tz-writer.md) | После сбора контекста | Пишет ТЗ по схеме с однозначными требованиями |
| [TZ_Reviewer](tz-reviewer.md) | После написания ТЗ | Проверяет по чеклисту из 15 пунктов, возвращает список проблем |

## Пайплайн

```
Задача → TZ_Context_Gatherer → резюме контекста
       → TZ_Writer → черновик ТЗ
       → TZ_Reviewer → список проблем → доработка → готовое ТЗ
```

TZ_Reviewer также работает автономно — можно скормить ему чужое ТЗ (от заказчика, коллеги) и быстро найти дыры без прочтения всего документа.
