# PERSONAL — Личная AI-продуктивность

## Мои скилы

Формат: `[имя скилла] — когда триггерится — что делает — какой контекст подкладывает`

**MCP-коннекторы:**

- `[Google Calendar]` — во время weekly review и планирования — читает события из GCal, считает часы по категориям, подставляет данные в шаблон ретро — подкладывает реальное расписание недели

**Vault-команды (`.claude/commands/`):**

- `[create-note]` — когда нужно зафиксировать новое знание — создаёт атомарную заметку в `1-knowledge/notes/` — подкладывает схему frontmatter, правило "одна тема = один файл", структуру разделов
- `[create-playbook]` — когда нужно задокументировать повторяемый процесс — создаёт пошаговый плейбук в `1-knowledge/playbooks/` — подкладывает RIPO-шаблон (Role / Input / Process / Output) и требования к исполняемости
- `[create-contact]` — при знакомстве с новым человеком — добавляет запись в личный CRM в `1-knowledge/contacts/` — подкладывает схему контакта с полями expertise, met, category, history
- `[process-meeting]` — после встречи или созвона — превращает транскрипт/заметки в структурированный meeting note — подкладывает шаблон: Summary / Key decisions / Action items / Open questions
- `[weekly-review]` — каждое воскресенье — проводит спринт-ретро: закрывает неделю, обновляет OKR, ставит 3 SMART-цели на следующую — подкладывает текущие OKR, шаблон SMART, итоги прошлых недель
- `[start-project]` — при старте нового проекта — создаёт структуру директорий с README, tasks.md, CLAUDE.md — подкладывает vault map и правила именования

**Плагины Claude Code** — сторонние пакеты скилов, устанавливаемые в Claude Code как расширения. Каждый плагин добавляет набор специализированных скилов с заранее прописанными инструкциями, контекстом и процессами. Я использую три плагина:

*Superpowers* — процессные скилы, которые задают дисциплину работы:
- `[superpowers:brainstorming]` — перед реализацией любой фичи или документа — исследует намерение, требования, дизайн — не даёт начать реализацию без понимания "зачем"
- `[superpowers:systematic-debugging]` — при любом баге или неожиданном поведении — строго формулирует гипотезу, проверяет, фиксирует — не даёт гадать без доказательств
- `[superpowers:writing-plans]` — перед многошаговой реализацией — строит детальный план с файлами и шагами — защищает от непродуманной реализации
- `[superpowers:test-driven-development]` — при любой реализации фичи или фикса — сначала тест, потом код — подкладывает принцип red/green/refactor

*Frontend Design* — специализированный плагин для UI-разработки:
- `[frontend-design:frontend-design]` — при создании любого веб-интерфейса — генерирует production-grade UI с высоким качеством дизайна — подкладывает дизайн-принципы, паттерны компонентов и избегает generic AI-эстетики

*Feature Dev* — плагин для проектирования и реализации фич:
- `[feature-dev:feature-dev]` — при разработке новой функциональности — анализирует кодовую базу, строит архитектуру, создаёт план реализации — подкладывает паттерны существующего кода и конвенции проекта

(Код vault-команд — в [skills/](skills/). Плагины устанавливаются через Claude Code.)

## Задачи, под которые планирую найти/собрать скилы

### Meta-скилы (универсальные, переиспользуются в любых проектах)

- **Site_Monitor** — вручную обхожу десятки сайтов в поисках новых анонсов, трачу часы в неделю — хочу автоматический мониторинг любого списка сайтов с дайджестом обновлений
- **PDF_Rule_Extractor** — вручную читаю PDF-документы вузов и регуляторов, чтобы вытащить нужные правила — хочу автоматическое извлечение структурированных данных из любых PDF
- **Market_Researcher** — каждый раз заново ищу конкурентов и игроков рынка с нуля — хочу глубинный поиск с классификацией по категориям за один запрос
- **Funding_Monitor** — не успеваю следить за грантами и субсидиями, узнаю о них случайно — хочу мониторинг всех источников финансирования по заданной теме
- **Profile_Mapper** — вручную собираю и структурирую данные о человеке из разных источников — хочу автоматическое преобразование достижений в карту навыков
- **Strategy_Generator** — трачу время на ручной подбор опций под профиль человека — хочу скил, который по профилю + базе вариантов генерирует готовый план
- **Pitch_Builder** — каждый раз переписываю питч под новый формат с нуля — хочу генерацию под любой формат (холодный аутрич, one-pager, структура презентации) по одному описанию проекта

### Скилы для private-education (domain-specific, строятся поверх meta)

- **Career_Event_Monitor** — не успеваю отслеживать дни открытых дверей по всем вузам вручную — `Site_Monitor` на образовательных платформах с еженедельным дайджестом мероприятий
- **Olympiad_Tracker** — дедлайны олимпиад разбросаны по десяткам сайтов, легко пропустить — мониторинг РСОШ и олимпиадных порталов с уведомлениями о приближении сроков
- **BVI_Rule_Aggregator** — правила БВИ меняются каждый январь, читать PDF каждого вуза вручную нереально — `PDF_Rule_Extractor` для сопоставления дипломов олимпиад с правами по факультетам
- **Student_Skill_Mapper** — вручную пишу карту навыков каждого ребёнка после интервью — `Profile_Mapper` для школьников с визуальными схемами под шаблоны дизайна
- **Grant_&_Subsidy_Finder** — пропускаю гранты для образовательных проектов из-за разрозненных источников — `Funding_Monitor` по теме образования и талантливой молодёжи
- **Education_Market_Deep_Researcher** — рекомендательная база кружков устаревает, обновлять вручную долго — `Market_Researcher` по EdTech с классификацией по секторам
- **Personalized_Strategy_Generator** — составление индивидуального плана ребёнка занимает несколько часов работы — `Strategy_Generator` по профилю ребёнка + базе активностей
