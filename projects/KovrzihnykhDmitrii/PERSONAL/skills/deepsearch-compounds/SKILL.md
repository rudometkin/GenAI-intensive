# deepsearch-compounds

## Когда триггерится
- пришло название органики/пищевого компонента/E-кода/CAS-номера;
- пользователь спрашивает «природное или синтетическое»;
- пользователь спрашивает «есть ли оно в <продукт>»;
- пользователь спрашивает про безопасность, ПДК, ADI, канцерогенность.

## Что делает
Собирает по одному и тому же соединению три независимых среза и сверяет их между собой:

1. **Химическая природа** — структура, CAS, синонимы, биосинтез/синтез, где встречается в природе (растения, животные, грибы, бактерии).
2. **Присутствие в еде** — в каких продуктах содержится как естественная компонента; в каких — как добавка/контаминант; типичные концентрации.
3. **Безопасность для человека** — ADI (WHO/JECFA), NOAEL, регуляторный статус (EFSA, FDA, Роспотребнадзор, EAEU), острые и хронические эффекты, группы риска (беременные, дети, аллергики).

На выходе — структурированная карточка соединения + явно отмеченные противоречия между источниками.

## Какой контекст подкладывает
- список предпочтительных источников: PubChem, PubMed, EFSA opinions, JECFA monographs, FDA GRAS, EAEU TR CU 029/2012, Роспотребнадзор СанПиН;
- чек-лист полей карточки (см. ниже);
- правило «синтетическое = идентично природному ≠ безопаснее» — не путать;
- правило «дозозависимость обязательна» — без упоминания дозы выводов не делать;
- шаблон disclaimer-а: «не является медицинской консультацией».

## Шаблон выходной карточки

```
# {Название соединения}
## Идентификаторы
- IUPAC: ...
- CAS: ...
- Синонимы / торговые названия: ...
- E-код (если пищевая добавка): ...

## Природа
- Природное / синтетическое / nature-identical
- Где встречается в природе: ...
- Как получают промышленно: ...

## Где встречается в еде
- Естественно (с типичными концентрациями): ...
- Как добавка (технологическая функция): ...
- Как контаминант: ...

## Безопасность
- ADI / TDI (источник, дата): ...
- NOAEL: ...
- Регуляторный статус: EFSA / FDA / EAEU / РФ
- Острая токсичность: ...
- Хронические эффекты, канцерогенность (IARC группа, если есть): ...
- Группы риска: ...

## Противоречия в источниках
- ...

## Вердикт по конкретному вопросу пользователя
- ...

## Disclaimer
Это информация из открытых источников, не медицинская консультация.
```

## Алгоритм исследования
1. Распознать сущность (название → CAS → E-код).
2. Достать химическую карточку из PubChem.
3. Проверить регуляторный статус в EFSA + FDA + EAEU TR CU 029/2012.
4. Достать ADI/NOAEL из JECFA/EFSA.
5. Проверить наличие в конкретном продукте, если он указан (базы USDA FoodData Central, Phenol-Explorer).
6. Собрать карточку, явно выписать противоречия, дать вердикт на конкретный вопрос пользователя.
7. Подписать источники с датами.

## Чего скилл НЕ делает
- не даёт медицинских рекомендаций;
- не ставит диагноз;
- не отвечает «безопасно / не безопасно» без указания дозы и группы;
- не ссылается на просроченные оценки без пометки даты.
