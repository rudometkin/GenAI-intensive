# find-hosting-ru

## Когда триггерится
- нужен хостинг / облако / VPS / S3-совместимое хранилище / managed БД / CDN / Kubernetes;
- упоминается ограничение: «оплата из РФ», «карта МИР», «без VPN», «под 152-ФЗ», «ПД в РФ»;
- вопрос «можно ли ещё пользоваться <провайдер>»;
- вопрос «альтернатива для <стек> / <регион>».

## Что делает
Возвращает **таблицу 5–10 кандидатов** с одинаковым набором колонок, так чтобы решение было сравнимым, а не «пойдём в Selectel, потому что кто-то посоветовал».

Колонки таблицы:

| Провайдер | Тип (VPS/облако/PaaS) | Доступность из РФ | Способы оплаты из РФ | Юрисдикция / где ПД | Ценник на эталонную конфигурацию | SLA | Риски / ограничения | Удобство миграции |
|---|---|---|---|---|---|---|---|---|

После таблицы — явный **вердикт по trade-off-ам** в 3–5 строк и рекомендация под конкретный кейс.

## Какой контекст подкладывает
- список провайдеров с локальным присутствием (Yandex.Cloud, VK Cloud, Selectel, Timeweb Cloud, Cloud.ru / SberCloud, Rusonyx, Reg.Ru, FirstVDS, Beget, MCS-like);
- список «серых зон» и «ушедших» (AWS, GCP, Azure, DigitalOcean, Vercel, Netlify, Cloudflare Workers — что работает, что нет, как оплачивать);
- список нейтральных / через третьи страны (Hetzner, Contabo, OVH, UpCloud, Scaleway — что с картами, что с санкционкой);
- требования 152-ФЗ (ПД граждан РФ хранить в РФ, регистр операторов ПД);
- требования на инфо-системы (ФСТЭК, ГОСТ-криптография, если продукт для госа);
- шаблон «эталонная конфигурация» для сравнения (vCPU / RAM / NVMe / трафик).

## Эталонные конфигурации для бенчмарка
Сравнивать кандидатов нужно на одних и тех же SKU:

1. **small-web**: 2 vCPU / 4 GB RAM / 40 GB NVMe / 2 TB трафика — для MVP, пет-проекта, лендинга.
2. **api-prod**: 4 vCPU / 8 GB / 80 GB NVMe / 4 TB / managed Postgres 50 GB.
3. **ml-light**: 8 vCPU / 32 GB / 200 GB / GPU T4 или аналог.
4. **s3-cold**: 1 TB S3-совместимого object storage, redundancy class standard.

## Алгоритм
1. Уточнить у пользователя:
   - стек (язык, фреймворк, БД, статика/SSR);
   - есть ли ПД граждан РФ;
   - кто платит: юрлицо РФ, физик РФ, юрлицо за рубежом;
   - критичность SLA (pet / MVP / prod / регламентированный сектор);
   - планируемая нагрузка и география пользователей.
2. Выбрать 2 эталонных SKU из списка выше, которые ближе всего.
3. Для каждого кандидата заполнить таблицу. Источники — только официальные price-листы провайдера.
4. Отдельно отметить, кто умеет принимать карты МИР / ЮMoney / расчётный счёт РФ.
5. Выписать «желтые флаги»: удалённые регионы, деградация SLA, публичные инциденты за последние 12 мес.
6. Дать рекомендацию под кейс, с 2–3 альтернативами второго эшелона.

## Правила скилла
- никогда не советовать провайдера, про которого известно, что он закрывает аккаунты из РФ, без явного warning-а;
- никогда не предлагать «заплатить через друзей / прокладки» — это техдолг и риск блокировки;
- отдельно выделять кейс «ПД граждан РФ» → только провайдер с ЦОД в РФ;
- если нагрузка нерегламентированная и платёж возможен через нерезидентское юрлицо — честно показать, что Hetzner/Contabo в 3–5 раз дешевле Selectel на том же SKU.

## Что НЕ делает
- не даёт окончательных юридических заключений по 152-ФЗ — отмечает риск и советует юриста;
- не делает маркетинговых обзоров «топ-10 лучших хостингов» — только под конкретный кейс;
- не скрывает недостатки провайдера, даже если у него хорошее имя.
