# project-search

> Это бонусный скилл из задания «5» — пошаговый набор для поиска проектов под запрос со сравнением **не менее 20** сервисов за прошедший год и выбором наиболее подходящего. Здесь реализован именно шаг «поиск и сравнение»; остальные шаги (анализ логики, ТЗ, реализация, тесты, публикация) — отдельные скиллы.

## Когда триггерится
- «кто уже делает <X>», «есть ли уже такое»;
- «найди похожие сервисы», «аналоги», «конкуренты»;
- «подбери кандидатов для PROJECTCLONE» / «что клонировать в тематике <X>»;
- перед принятием решения «делать / не делать» новый продукт;
- перед переговорами с инвестором / клиентом, когда нужна карта рынка.

## Что делает (коротко)
1. Разворачивает запрос в **query-pack** (5–10 перефразированных формулировок для разных каналов поиска).
2. По query-pack проходит по **6 каналам** и собирает длинный список кандидатов. Целевое значение — **≥20** уникальных сервисов за последние 12 месяцев.
3. Оценивает каждого по **единой рубрике** (7 осей).
4. Возвращает **сравнительную таблицу 20+** и **топ-3 чемпиона** под исходный запрос с обоснованием и трейдоффами.

## Какой контекст подкладывает
- список обязательных каналов поиска (см. ниже);
- шаблон query-pack;
- единая рубрика оценки;
- правила дедупликации («один продукт — одна строка»);
- правило «активности за 12 месяцев»;
- формат таблицы и финального вывода.

## Алгоритм

### Шаг 1. Уточнить запрос (30 секунд разговора)
Скилл обязан знать минимум:
- **Problem statement** в одной строке: «помочь <кому> сделать <что> в <ситуации>».
- **Регион пользователя и биллинга** (важно: карты / язык интерфейса / юрисдикция).
- **Цена времени**: «надо за 1 час» vs «надо сравнение для инвестора на 2 недели».
- **Что считать «подходящим»**: цена / качество / скорость / наличие API / наличие в РФ.

Если что-то не сказано — спросить до поиска, а не после.

### Шаг 2. Query-pack
Из problem statement собрать 5–10 формулировок. Пример:
- прямая: `language learning with short videos`
- роль пользователя: `app for immigrants to learn spoken <lang>`
- механика: `tiktok style language learning`
- близкие тэги: `comprehensible input app`, `shadowing app`
- косвенная: `how do I learn <lang> by watching clips`
- синонимы на 2-м языке: `приложение для изучения <lang> по видео`
- сленг/мем: `doomscroll language`

### Шаг 3. Каналы поиска (все 6 — обязательные)
| # | Канал | Что даёт | Как искать |
|---|---|---|---|
| 1 | **Product Hunt** | свежие запуски, reviews, альтернативы | поиск по query-pack + раздел `Alternatives` у найденных |
| 2 | **Y Combinator / batch-lists / a16z portfolio / openVC** | «backed by» и фаундерский уровень, signalling-серьёзность | YC companies filter по тегу + поиск по Crunchbase |
| 3 | **G2 / Capterra / TrustRadius** | B2B-сегмент, честные отзывы, ценники | категория + leaderboard + вкладка `Alternatives` |
| 4 | **GitHub trending / awesome-lists** | опенсорсные клоны, self-hosted, наборы ссылок | поиск по query-pack + `awesome-<topic>` репо |
| 5 | **TechCrunch / The Verge / Wired / VC.ru / Habr** | посевы, Series A, контекст | site-search + дата за последние 12 мес |
| 6 | **Соцсети и ресёрчеры** | реальные пользователи, UGC-плейлисты, Reddit, X-треды, Telegram-каналы | Reddit /r/<topic>, X search, тематические Telegram-папки |

Минимум **по 3–5 кандидатов из каждого канала**, итого ≥20 уникальных.

### Шаг 4. Дедупликация и фильтр «живой за 12 мес»
Отфильтровать:
- мёртвые (нет релизов/постов/твитов 12+ месяцев, сайт 404, LinkedIn «closed»);
- дубли (один бренд, разные домены);
- не-прямые аналоги (другой problem statement) — но вынести их в `adjacent/`.

На выходе: ≥20 активных строк.

### Шаг 5. Единая рубрика оценки (7 осей)
Каждому сервису проставить балл 1–3 по каждой оси:

| Ось | 1 | 2 | 3 |
|---|---|---|---|
| **problem-fit** | закрывает часть | закрывает большинство | полностью закрывает problem statement |
| **moat** | нет (обёртка над чужим API) | частичный (данные/бренд) | явный (сеть, регуляторка, exclusive data) |
| **traction** | pre-product / Alpha | есть платящие | растущая MRR / >100k MAU |
| **UX** | неудобно | ок | выделяется на фоне категории |
| **cost to user** | дорого для ЦА | медиана | дешевле рынка / freemium честный |
| **access (регион пользователя)** | не работает / не платится | работает через костыль | без трения |
| **defensibility vs мой проект** | легко обойти | придётся постараться | почти невозможно догнать |

Сумма 7–21.

### Шаг 6. Таблица
```
| # | Сервис | Страна / год | 1-линейник | Модель | Ценник | Регион/оплата | problem-fit | moat | traction | UX | cost | access | def | ИТОГО | ссылки |
```

### Шаг 7. Топ-3 и вердикт
Не «победитель по сумме», а 3 кандидата под **разные трейдоффы**:
1. **Эталон-чемпион** — лучший по problem-fit + UX, без скидок. Его разбираем как best practice.
2. **Самый опасный конкурент для моего проекта** — наивысший `def` или траекция, даже если UX слабее.
3. **Самый удобный клон** — дешёвый MVP с понятными модулями, на котором тренируется команда.

Для каждого — 3 строки: «что забираем», «что не повторяем», «где их слабое место».

### Шаг 8. Мета-выводы
После таблицы — 5–10 строк:
- насыщенность категории (green / yellow / red);
- где ещё нет доминирующего игрока («белое пятно»);
- какие именно рвы уже застолблены и какие остались свободными;
- какое окно по времени остаётся.

## Правила скилла
- **минимум 20 живых кандидатов** — если не набралось, сказать честно и не заполнять таблицу выдуманными именами;
- **единая рубрика обязательна** — нельзя оценивать одного по features, другого «по ощущениям»;
- **ссылка + дата последнего релевантного сигнала** у каждой строки;
- **trade-off прежде всего** — топ-1 «по сумме» без логики почему не выдаётся;
- не сваливать в маркетинговый обзор «топ-10 лучших приложений» — это не цель скилла;
- adjacent-категории выносить в отдельный блок, не мешать с прямыми конкурентами;
- не сравнивать одного по старым данным (2021), а другого — по свежим — всё в окне 12 месяцев.

## Что скилл НЕ делает
- не анализирует UX/логику интерфейса «в микроскоп» — это отдельный скилл (`interface-deep-dive`);
- не пишет ТЗ на клон — это скилл `tz-writer`;
- не предлагает имплементацию — это следующий шаг бонус-трека.

## Связь с другими скиллами бонус-трека
