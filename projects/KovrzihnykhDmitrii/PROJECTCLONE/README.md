# PROJECTCLONE — vniklay

Клон-демо сервиса для изучения языка через **думскролл** коротких видео с двуязычными субтитрами (RU/EN). Рабочее название — **vniklay**.

Репозиторий кода: [vniklay-main.zip](./vniklay-main.zip) (Vite + React + TypeScript + Tailwind, `src/components/ImmersiveViewer.tsx`, `src/data/scenes.ts`, 10 демо-видео).

---

## 1. Почему выбрали этот сервис

Изучение языка проваливается не на «недостатке контента», а на **отсутствии привычки**. У людей уже есть устойчивая привычка — думскроллить TikTok/Reels/Shorts. Если перенести механику бесконечной вертикальной ленты на контент из **настоящей разговорной речи носителей**, учащийся получает:

- пассивное насмотренное + прослушанное (шэдоуинг),
- живые идиомы и интонации, которых нет в учебниках,
- микродозы по 20–60 секунд, которые не требуют силы воли.

То есть мы клонируем не «Duolingo», а формат-оболочку TikTok и начиняем её обучающим ядром. Похожие идеи есть у **FluentU**, **Lingopie**, **Chai**, **Jumpspeak**, **Speechling**, но ни один из них пока не даёт честный вертикальный ленточный UX с дуальными субтитрами «сразу из коробки» и без подписки на первом шаге.

## 2. Какие скиллы нужны были для сборки решения

| Скилл | Что делает | Где применён в демо |
|---|---|---|
| `viral-clip-picker` | находит и оценивает виральность коротких клипов, подбирает под уровень языка | выбор 10 исходных видео |
| `subtitle-maker` | генерация двуязычных субтитров (ASR → MT → выравнивание тайминга) | `scenes.ts` с полями `start/end/ru/en` |
| `frontend-builder` | собирает одностраничный вертикальный viewer (swipe, wheel, touch, мьют, плей/пауза) | `ImmersiveViewer.tsx` |
| `social-pr` | делает посевы в TikTok/Reels/Telegram, пишет хуки и копирайт | (на демо не применён, только сформулирован) |

Дополнительно для продакшена понадобятся: `legal-clearance` (права на клипы), `level-classifier` (A1–C2), `srs-memory` (повторение лексики), `community-moderation` (UGC-субтитры).

## 3. На какие продуктовые модули раскладывается функционал

| Модуль | Ответственность | Как закрывается скиллами |
|---|---|---|
| **Фронт / Player** | Immersive viewer, жесты, субтитры поверх видео, toggle RU/EN, таймлайн, «сохранить фразу» | `frontend-builder` + `subtitle-renderer` |
| **Бэк / API** | Выдача карточек сцен, прогресс пользователя, аналитика событий (view, replay, save_phrase) | `backend-api` + `events-logger` |
| **Хранилище** | Видео (CDN/S3), субтитры (JSON), словарь фраз, профиль пользователя | `content-storage` + `profile-service` |
| **Контент-инжест** | Добыча исходников, нарезка, ASR, перевод, выравнивание, модерация | `viral-clip-picker` + `subtitle-maker` + `level-classifier` |
| **Рекомендательная** | Подбор следующего видео под уровень, интересы, прогресс; diversity/novelty | `ranker` + `taste-profile` (пересечение с идеями Food2Mood) |
| **Рост** | Посевы, SEO по фразам, referral | `social-pr` + `growth-hacking` |

## 4. Загруженный код

См. `vniklay-main.zip` в этой папке (можно распаковать и запустить `npm i && npm run dev`). В `src/data/scenes.ts` — 10 сцен с разметкой субтитров; компонент `ImmersiveViewer` реализует swipe-навигацию, мьют, двойные субтитры.

## 5. Логика: как создавали, что было сложным, какие вопросы остались

**Как создавали.** Разделили работу на 4 потока параллельно: контент (10 клипов), субтитры (ASR + ручная редактура, дуальные RU/EN), UI (вертикальный viewer с плавным переходом между клипами), пиар-черновик. Собрали демо на Vite+React, чтобы быстро показать UX.

**Что было сложным.**
1. **Тайминг субтитров** — ASR дал грубый транскрипт, тонкий тайминг под 0.2 сек пришлось редактировать руками.
2. **Смешанные языки в одном клипе** (видео 6, 8 и 10 — англоязычные отрывки с RU-переводом) — пришлось держать `ru` и `en` симметрично, чтобы тот же рендерер работал в обе стороны.
3. **Жесты на десктопе и мобильнике одновременно** (touch / wheel / click) без конфликтов — ввёл cooldown на `wheel`.
4. **Права на видео** — для демо использовали публично доступные куски интервью. Для продакшена это главный блокер.

**Открытые вопросы.**
- Как юридически чисто собирать контент в масштабе? Лицензировать у рилс-крейторов? Собирать собственный (снятый под задачу)?
- Какой сигнал считать «понял фразу»: пауза, rewind, save, вручную «не понял»?
- Делать ли UGC-субтитры (как Lingq)? Где грань между скоростью контента и качеством?
- Уровень языка на клипах: размечать вручную, моделью или по реакциям пользователей?

## 6. ⭐ Как построить ров вокруг vniklay (разного типа)

### 6.1 Ров по данным (data moat)

**Идея.** Накопить уникальный корпус: `(клип, сегмент, фраза, уровень, реакция пользователя)`. Чем больше пользователей, тем точнее модель уровня фразы и подбор следующего клипа — и тем сложнее конкуренту догнать качество ленты.

Как строить:
- логировать `view / long_view / replay / pause_at / save_phrase / skip / mark_unknown` на уровне сегмента субтитра;
- строить вкусо-уровневый профиль пользователя (пересечение с подходом из Food2Mood — sparse-профиль по тегам: `grammar:past_simple`, `topic:relationships`, `pace:fast`, `accent:US`);
- через 3–6 месяцев использовать этот корпус для своей модели ранжирования, которой нет у конкурентов.

Метрика рва: `median time-to-first-save`, `% retained at D7`, доля сегментов с ≥100 сигналов.

### 6.2 Ров по доверию / контенту (trust + editorial)

**Идея.** Курация от сертифицированных преподавателей / лингвистов. Не автоматические субтитры «как есть», а выверенные, с пометками «идиома», «сленг», «редкая конструкция». Такую метку не воспроизведёт ни один скрейпер TikTok.

- Нанять 3–5 редакторов под каждый языковой поток на первые 1000 клипов.
- Публичные профили преподавателей → перенос доверия на бренд.
- Бейдж «verified clip» на карточках.

Метрика рва: доля verified-клипов, NPS среди B2B-клиентов (школы языков).

### 6.3 Ров по переключению (switching cost)

**Идея.** Учащийся не уйдёт, если его прогресс живёт внутри сервиса и имеет цену.

- Персональный словарь фраз, собранный свайпом → SRS-повторения (как Anki, но автоматически).
- История «какие клипы досматривал, на каких залипал» — собственная сводка прогресса.
- Сертификаты уровня, которые можно показать работодателю / вузу.
- Экспорт фраз в форматах Anki/Quizlet — парадоксально, но это **увеличивает** привязку, потому что пользователи делают вложение в инструмент.

Метрика рва: размер персонального словаря у Mx пользователя, `retention at D30/D90`.

### 6.4 Ров по сети (network effect)

**Идея.** UGC и социальные сигналы.

- Пользователи могут предлагать свои клипы (с модерацией) и альтернативные переводы/заметки.
- Публичные плейлисты: «30 клипов для собеседования», «фразы из Peaky Blinders».
- Ссылки на конкретный сегмент (`/clip/2165?t=15.48`) работают как вирусная единица — их шарят в Telegram/Twitter, каждая шара приводит нового пользователя.

Метрика рва: `viral coefficient k`, доля UGC-плейлистов в top-100.

### 6.5 Регуляторный / юридический ров

**Идея.** Официальные сделки с правообладателями (студии, инфлюенсеры, образовательные каналы). Один раз сделать больно (переговоры, лицензии), зато закрыть вход конкурентам — повторить дорого.

- Сначала покрыть англоязычный lifestyle/interview контент.
- Дальше — локальные лицензии (сербский, турецкий, армянский — для эмигрантов).
- «License & Revenue-share» как модель для контент-партнёров.

Метрика рва: количество эксклюзивных партнёрств, % лицензированного контента в ленте.

### 6.6 Бонус: ров через встроенность (distribution moat)

**Идея.** Интеграция с реальными курсами (Skyeng, Puzzle English, школы в СНГ) как «рекомендованный инструмент для тренировки слуха». Это даёт дистрибуцию и B2B2C-деньги на старте и формирует барьер: конкурент должен не только сделать продукт, но и выбить у школы замену.

Метрика рва: ARR из B2B-канала, количество ежедневных SSO-сессий из школ.
