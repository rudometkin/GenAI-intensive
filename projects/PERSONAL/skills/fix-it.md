# fix-it

**Триггер:** трейсбек в консоли / «не работает» / «ошибка X» / «сломалось»

## Что делает

1. Читает трейсбек и определяет виновный файл и строку
2. Читает этот файл вокруг проблемного места
3. Проверяет `decisions/log.md` — нет ли известной причины
4. Предлагает минимальный фикс (не рефакторинг, не очистка — только суть)
5. После фикса проверяет, не сломало ли смежное

## Контекст

```
Из трейсбека → читать виновный файл
Всегда проверять: kbzhuy/docs/decisions/log.md
При ошибках миграций: backend/migrations/versions/
При ошибках API: backend/app/api/v1/ + backend/app/schemas/
При ошибках mobile: mobile/src/api/ + соответствующий store
```

## Соглашения

- Фиксировать только то, что сломано — не трогать остальное
- Bat-файлы: не добавлять `>nul` (заменяется на `>/dev/null` линтером)
- Mobile preview: `expo export` → `python serve.py` (не Metro — нестабилен с кириллическим путём)
