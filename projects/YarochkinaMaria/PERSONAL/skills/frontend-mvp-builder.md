# Frontend MVP Builder

## Статус
Используется сейчас

## Когда триггерится
- Нужен рабочий прототип для демо, валидации идеи или учебного задания
- Хочу показать продукт стейкхолдерам — без бэкенда, но с реальным UI
- Готов набор модулей и user flow, нужен код

**Фразы:** «собери MVP для [сервис]», «сгенерируй frontend», «создай рабочий прототип», «напиши код для MVP»

## Что делает
1. Генерирует структуру проекта: Next.js 14 + TypeScript + Tailwind CSS
2. Реализует все страницы по модулям и user flow
3. Создаёт state management (localStorage / React Context — без бэкенда)
4. Добавляет мокданные для демо
5. Строит компоненты: навигацию, статусы, карточки, формы
6. Применяет mobile-first дизайн с профессиональной палитрой

## Входные данные
- Описание продукта + JTBD (обязательно)
- Модульная декомпозиция (желательно — даёт точнее результат)
- Дизайн-референсы или палитра (опционально)
- Технический стек (опционально, дефолт: Next.js + Tailwind)

## Что выдаёт на выходе
Полный работающий frontend: все страницы, компоненты, логика переходов, мокданные. Готов к деплою на Vercel.

## Как помогает мне в работе
Позволяет перейти от идеи к демо за один сеанс. Критически важно для валидации гипотез и показа концепта без привлечения разработчиков.

## Пример реального использования
Для Settld-клона: из модульной декомпозиции + user flow → за одну сессию получила полный MVP с 8 страницами, state machine на 6 статусов и демо-данными.
