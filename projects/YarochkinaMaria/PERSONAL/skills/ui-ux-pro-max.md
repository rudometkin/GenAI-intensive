# UI/UX Pro Max

## Статус
Используется сейчас

## Когда триггерится
- Разрабатываю или проверяю UI — нужно сделать профессионально
- Выбираю цвета, типографику, стиль для продукта
- Хочу проверить UX на соответствие стандартам (accessibility, touch, responsive)
- Проектирую новую страницу или компонент

**Фразы:** «сделай красивый UI», «выбери цвета», «это выглядит непрофессионально», «улучши дизайн», «добавь mobile-first», «проверь доступность»

## Что делает
Даёт конкретные рекомендации по дизайну из базы: 50+ стилей, 161 цветовая палитра, 57 шрифтовых пар, 99 UX-правил. Проверяет по приоритетной матрице: accessibility → touch → performance → style → layout → typography.

## Входные данные
- Описание задачи: что проектируем (страница, компонент, стиль)
- Целевая аудитория и тип продукта
- Технический стек (React, Next.js, Tailwind, Swift и т.д.)
- Референсы или текущий код (опционально)

## Что выдаёт на выходе
Конкретные рекомендации: цветовая палитра с hex-кодами, шрифтовая пара, правила отступов и типографики, код компонентов, флаги нарушений (contrast ratio, touch target size, etc.).

## Как помогает мне в работе
Закрывает дизайн-компетенцию без дизайнера. Для PM, который строит MVP или прототип — критично не облажаться с визуалом на демо.

## Пример реального использования
Для Settld MVP: палитра #1C3D5E (тёмно-синий) + #F5F5F3 (тёплый белый), Inter, мобильные карточки с shadow-card, цветовая система для 6 статусов запросов — всё по правилам accessibility.
