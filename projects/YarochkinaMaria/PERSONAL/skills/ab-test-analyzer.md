# A/B Test Analyzer

## Когда триггерится

- Тест завершён — нужен вывод: запускать, откатывать, итерировать
- Результаты «на грани» статзначимости — непонятно что решать
- Нужно объяснить результат стейкхолдерам нетехническим языком
- Подозрение на novelty effect, seasonal bias или segment dilution

**Триггерные фразы:** «тест закончился», «результаты теста», «p-value = ...», «что значат эти данные», «анализ A/B», «стоит ли запускать»

---

## Что делает

1. Принимает сырые данные теста (или текстовое описание)
2. Проверяет корректность: размер выборки, длительность, метрика, сегментация
3. Интерпретирует статистическую значимость и практическую значимость (effect size)
4. Флажит типичные ошибки: peaking, multiple comparisons, SRM (Sample Ratio Mismatch)
5. Строит итоговый вывод: LAUNCH / HOLD / ITERATE + обоснование
6. Формулирует текст для синхронизации с командой

---

## Входные данные

- Метрика + baseline
- Результаты: конверсия/значение в control и test, N пользователей
- Длительность теста
- p-value (если есть) или сырые числа (посчитает сам)
- Контекст: что меняли, где в воронке

---

## Выход

```
A/B TEST REPORT
───────────────
Тест: [название]
Результат: LAUNCH ✅ / HOLD ⏸ / ITERATE 🔁
────────────────────
Control: [метрика] = X%  (N = ...)
Test:    [метрика] = Y%  (N = ...)
Lift: +Z% (relative)
p-value: 0.0X | Статзначим: да/нет
Effect size: [оценка практической значимости]
────────────────────
Флаги: [SRM? Peaking? Seasonal?]
Вывод: [1-2 предложения]
Следующий шаг: [конкретное действие]
```

---

## Почему полезен именно мне

В работе с growth и конверсией результаты тестов часто интерпретируются слишком быстро или слишком осторожно. Этот skill даёт мне надёжный второй голос при принятии решений по тестам и экономит 30–60 минут на каждом анализе.
