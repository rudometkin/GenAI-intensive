# JTBD Extractor

## Статус
Используется сейчас

## Когда триггерится
- Нужно понять что реально делает пользователь в продукте — за рамками фич
- Формулирую core scenario для MVP или нового направления
- Хочу проверить: то, что мы строим — реально нужный job?

**Фразы:** «выдели JTBD для [сервис]», «найди core job-to-be-done», «какой главный сценарий», «что делает пользователь»

## Что делает
1. Анализирует продукт (через WebSearch / WebFetch если нужно)
2. Составляет список 3–7 JTBD в формате «Когда [ситуация], я хочу [действие], чтобы [результат]»
3. Выбирает одну главную JTBD — максимально конкретную и болезненную
4. Строит пошаговый user flow под эту JTBD
5. Разграничивает: что входит в MVP, что исключить

## Входные данные
- Название сервиса или описание продукта (обязательно)
- Контекст: полный анализ или под учебный MVP (опционально)
- Ограничения: что точно не делаем (опционально)

## Что выдаёт на выходе
Список JTBD с обоснованием выбора главного + пошаговый user flow (5–9 шагов) + список исключений из MVP.

## Как помогает мне в работе
Позволяет быстро перейти от «что это за продукт» к «что конкретно нужно построить». Особенно полезен при запуске нового продукта или оценке идеи стартапа — заменяет несколько часов ресёрча.

## Пример реального использования
Для Settld: из 5 возможных JTBD выделил главный — «уведомить все организации о смерти из одного места» — и построил под него 7-шаговый user flow для MVP.
