# Stakeholder Brief Writer

## Когда триггерится

- Нужно написать апдейт для C-level или совета директоров
- Готовится запрос на ресурсы / бюджет / приоритет
- Нужна синхронизация между командами с разными контекстами
- Приближается review, OKR-чекин, демо для руководства
- Есть плохие новости или риск — нужно корректно донести

**Триггерные фразы:** «напиши апдейт для», «оформи запрос на ресурсы», «нужен executive summary», «подготовь слайд для CEO», «синхронизация с...»

---

## Что делает

1. Определяет аудиторию и желаемый outcome (inform / decide / approve / align)
2. Выбирает нужный формат: memo / one-pager / slack update / meeting agenda
3. Строит структуру по принципу Pyramid Principle: вывод → аргументы → детали
4. Убирает воду, технический жаргон, детали которые аудитории не нужны
5. Добавляет чёткий call-to-action в конце
6. Адаптирует тон под аудиторию (CEO / совладелец / техлид / инвестор)

---

## Входные данные

- Тема: что нужно донести
- Аудитория: кто читает / слушает
- Желаемый outcome: что должна сделать аудитория после
- Ключевые факты и цифры
- Контекст: срочность, предыстория, что уже знают

---

## Выход

```
[ФОРМАТ: Memo / One-pager / Slack]
────────────────────────────────
SUMMARY (1-2 предложения: что происходит + что нужно)

CONTEXT
• [факт 1]
• [факт 2]

OPTIONS / SITUATION
• [вариант А / описание ситуации]
• [вариант Б]

RECOMMENDATION
→ [конкретное предложение с обоснованием]

NEXT STEPS
□ [кто / что / когда]
```

---

## Почему полезен именно мне

PM постоянно балансирует между техническим контекстом и бизнес-языком. Этот skill помогает за 5 минут превратить набор фактов в чёткий документ, который C-level прочитает и поймёт без вопросов. Особенно ценен при эскалации, запросах на ресурсы и кризисных коммуникациях.
