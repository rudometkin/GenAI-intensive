# Cloneability Assessment

## Статус
Используется сейчас

## Когда триггерится
- Нужно оценить сервис как кандидат для клонирования или конкурентного анализа
- Хочу понять: достаточно ли интересен продукт для глубокого разбора
- Сравниваю несколько сервисов и нужен системный способ выбрать лучший

**Фразы:** «оцени [сервис] для клонирования», «подходит ли [сервис] для учебного клона», «cloneability»

## Что делает
Оценивает сервис по 10 критериям (1–5 баллов каждый):
понятность сценария, ясность core value, декомпозируемость на модули, бизнес-логика, MVP-реалистичность, архитектурная описуемость, скилл-описуемость, потенциал moat-анализа, «умный выбор», риск масштаба.
Выдаёт честный вердикт: ОТЛИЧНЫЙ / ХОРОШИЙ / СПОРНЫЙ / ЛУЧШЕ ВЫБРАТЬ ДРУГОЙ.

## Входные данные
- Название сервиса (обязательно)
- Контекст: учебный клон, конкурентный ресёрч или стратегический анализ (опционально)

## Что выдаёт на выходе
Структурированный отчёт: описание сервиса, таблица оценок с комментариями, сильные стороны, риски, что упростить, как позиционировать, итоговый вердикт.

## Как помогает мне в работе
Убирает субъективность при выборе продуктов для анализа. Используется на этапе ресёрча — прежде чем тратить время на глубокое изучение сервиса.

## Пример реального использования
Оценила 20 кандидатов из сферы silver economy → Settld получил 49/50 → обоснованный выбор для PROJECTCLONE задания.
