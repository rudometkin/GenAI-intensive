# PERSONAL

Персональная AI операционная система product manager'а. Только реальные инструменты — те, что уже установлены и используются в работе.

---

## 1. Skills, которые я использую сейчас

Все 10 skills установлены в `~/.claude/skills/`. Файлы с документацией — в папке [`skills/`](./skills/).

---

**`cloneability-assessment`** — когда нужно оценить сервис как кандидат для анализа или клонирования — оценивает по 10 критериям (1–5), честно выдаёт вердикт и риски — подкладывает название сервиса и контекст цели — возвращает структурированный отчёт с рекомендацией

**`jtbd-extractor`** — когда нужно понять что реально делает пользователь в продукте или сформулировать core scenario — извлекает 3–7 JTBD, выбирает главный, строит пошаговый user flow — подкладывает описание продукта и контекст (полный анализ или MVP) — возвращает список JTBD + user flow + границы MVP

**`product-architecture-mapper`** — когда нужна декомпозиция перед roadmap-сессией или разработкой — раскладывает на 4–7 модулей, классифицирует core/supporting, предлагает три варианта scope — подкладывает описание продукта и JTBD — возвращает модульную карту + матрицу модуль→навыки

**`moat-designer`** — когда нужно оценить устойчивость продукта для pitch или стратегии — анализирует 7 источников рва, честно оценивает силу каждого, предлагает как укрепить — подкладывает описание сервиса, рынок и конкурентов — возвращает таблицу рва + топ-2 сильных + рекомендации

**`frontend-mvp-builder`** — когда нужен рабочий прототип для демо или валидации без разработчиков — генерирует Next.js + Tailwind фронтенд с мокданными, всеми страницами и логикой переходов — подкладывает модульную декомпозицию, user flow и стек — возвращает готовый к деплою проект

**`service-search-for-clone`** — когда нужно быстро понять ландшафт рынка в новой нише — ищет реальные существующие сервисы через WebSearch/WebFetch, группирует по категориям — подкладывает тему или рынок — возвращает структурированный список с описаниями

**`rapid-qa-reviewer`** — когда MVP готов и нужна проверка перед показом — проходит по user flow, читает код, находит сломанное и заглушённое, предсказывает неудобные вопросы — подкладывает путь к проекту и контекст аудитории — возвращает отчёт ✅/⚠️/❌ + топ-3 приоритетных фикса

**`ui-ux-pro-max`** — когда нужны конкретные дизайн-решения или проверка UI на профессиональность — выдаёт рекомендации по палитре, типографике, отступам, accessibility из базы 50+ стилей — подкладывает тип продукта, аудиторию и стек — возвращает готовые hex-коды, шрифты, правила компонентов

**`новый-скилл`** — когда удачный workflow нужно зафиксировать и переиспользовать — анализирует паттерн из текущего разговора, генерирует `SKILL.md` и устанавливает skill — подкладывает описание паттерна или историю сессии — возвращает установленный рабочий skill

**`PushToGenAI_public`** — когда нужно опубликовать материалы в репозиторий курса — проверяет безопасность, создаёт коммит с осмысленным сообщением, пушит в `rudometkin/GenAI-intensive` — подкладывает контент, путь и сообщение коммита — возвращает подтверждение и ссылку

## 2. Skills, которые пригодились бы в будущем

Следующий уровень — skills под ежедневную PM-работу, аналитику и собственные проекты.

---

**`growth-experiment-designer`** — нужен для оформления гипотез в A/B тесты — превращает «хочу попробовать X» в experiment brief с метрикой, выборкой и критериями успеха

**`ab-test-analyzer`** — нужен для интерпретации результатов тестов — читает данные, проверяет статзначимость, флажит SRM и novelty effect, выдаёт LAUNCH / HOLD / ITERATE

**`stakeholder-brief-writer`** — нужен для C-level коммуникации и запросов на ресурсы — превращает набор фактов в структурированный memo по Pyramid Principle

**`prd-writer`** — нужен для быстрого создания PRD и product spec — генерирует документ с проблемой, решением, метриками, scope и edge cases

**`conversion-funnel-analyzer`** — нужен для диагностики падения конверсии — находит drop-off, формулирует гипотезы причин, предлагает следующие шаги

**`sql-query-assistant`** — нужен для написания продуктовых SQL-запросов — генерирует запросы по описанию метрики (DAU, retention, funnel) под конкретную схему данных

**`research-synthesizer`** — нужен для обработки результатов интервью и опросов — синтезирует сырые данные в инсайты, паттерны, jobs и боли

**`elder-care-researcher`** — нужен для ресёрча рынка elder care и серебряной экономики — собирает данные по рынку, аудиториям, конкурентам, трендам и регуляторике по теме проекта Рядом

**`product-strategy-generator`** — нужен для структурирования стратегии и roadmap — превращает бизнес-цель в логику: метрика → инициативы → приоритизация → OKR

**`pm-career-packager`** — нужен для переупаковки опыта под конкретную роль — адаптирует резюме и LinkedIn под JD, строит единый карьерный нарратив

**`startup-pitch-builder`** — нужен для подготовки pitch deck и investment memo — строит нарратив: проблема → рынок → решение → traction → ask

**`high-trust-ux-designer`** — нужен для проектирования UX под уязвимые аудитории — правила и паттерны для пожилых, горя, стресса, медицинского контекста

**`c-level-communicator`** — нужен для подготовки к переговорам с топ-менеджментом — готовит executive summary, сценарии разговора, управление ожиданиями

**`okr-facilitator`** — нужен для формулировки и выравнивания OKR с командой — переводит стратегические цели в измеримые key results, проверяет на амбициозность и реалистичность
