# PROJECTCLONE — Клонирование InteriorAI

## 1. Выбранный сервис

**[InteriorAI](https://interiorai.com/)** — сервис AI-редизайна интерьеров: пользователь загружает фото комнаты, выбирает стиль, получает сгенерированный вариант интерьера.

Почему выбрал: InteriorAI — наиболее близкий аналог моего проекта Roomy. Интересен с точки зрения moat: сервис не работает с реальными каталогами мебели и не учитывает габариты — это и есть ключевое расхождение, которое Roomy закрывает.

## 2. Необходимые скиллы для сборки

- `react-components` — UI: загрузка фото, выбор стиля, отображение результата
- `backend-patterns` — API для обработки изображений и вызова модели
- `api-design` — REST API для генерации и хранения результатов
- `supabase-postgres-best-practices` — хранение фото пользователей и результатов
- `deployment-patterns` — деплой на Vercel + Railway

## 3. Продуктовые модули и покрытие скилами

| Модуль | Что делает | Покрывается скилом |

| Загрузка фото | Пользователь загружает фото комнаты | `react-components` |

## 4. Код проекта

Репозиторий: `/CODEBASE/` (MVP клона с базовым img2img через Replicate API)

Стек: Next.js 14, Supabase, Replicate API (stability-ai/stable-diffusion), Tailwind CSS, Vercel.

## 5. Логика

**Последовательность сборки:**
1. Создал Next.js проект с Supabase Auth (Google OAuth)
2. Подключил Replicate API — модель `stability-ai/stable-diffusion-img2img`
3. Построил UI: drag&drop загрузка фото, выбор стиля через карточки
4. Добавил хранилище результатов в Supabase Storage + таблица `generations`
5. Деплой на Vercel

**Что было самым сложным:**
- Подбор правильных промптов для img2img: без хорошего системного промпта модель ломает геометрию комнаты
- Баланс между strength параметром (насколько сильно менять фото) — слишком высокий разрушает пространство, слишком низкий не даёт результата

**Вопросы, которые остались:**
- Как сохранять перспективу и пространство при генерации — модель не понимает глубину
- Платная версия Replicate дорогая при масштабировании — нужна своя инфраструктура

## 6. ⭐ Ров

### Тип: Данные
InteriorAI не накапливает структурированные данные о комнатах. Ров: собирать размерные данные (площадь, высота потолков, планировка) вместе с фото → через 6 месяцев появляется датасет, который улучшает качество генерации под конкретные типы планировок.

### Тип: Switching cost
Пользователь один раз вводит параметры своей комнаты и сохраняет "комнату" в профиле. Повторные генерации не требуют повторного ввода → чем больше сессий, тем выше стоимость ухода.

### Тип: Сеть (каталог)
InteriorAI генерирует несуществующую мебель. Ров: партнёрство с реальными ретейлерами (Hoff, IKEA) → пользователь видит реальные товары с ценами и может купить одним кликом. Ретейлер платит за трафик → двусторонняя сеть.
