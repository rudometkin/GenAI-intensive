# PERSONAL — Личная AI-продуктивность

## Скилы, которыми пользуюсь

| Скилл | Когда триггерится | Что делает | Контекст |
|-------|-------------------|------------|----------|
| `react-components` | Создание нового UI-компонента | Генерирует компонент с правильной структурой, типами, стилями | Текущий стек (Next.js, Tailwind) |
| `backend-patterns` | Проектирование API эндпоинта | Строит слоистую архитектуру (router → service → repo) | Структура проекта |
| `api-design` | Проектирование публичного API | Генерирует OpenAPI-совместимый дизайн с правильными кодами и форматами | REST-конвенции |
| `supabase-postgres-best-practices` | Работа с БД: схемы, RLS, запросы | Пишет безопасные миграции, политики RLS, оптимальные запросы | Supabase |
| `market-research` | Исследование рынка / конкурентов | Структурированный анализ: сегменты, конкуренты, позиционирование | Отрасль проекта |
| `deep-research` | Глубокое исследование по теме | Многораундовый поиск с синтезом источников | Тема запроса |
| `security-review` | Перед деплоем / после изменений в auth | Проверяет уязвимости: XSS, SQLi, утечки ключей | Кодовая база |
| `deployment-patterns` | Настройка CI/CD, деплой | Генерирует конфиги Vercel/Railway, Docker, GitHub Actions | Стек деплоя |
| `tdd-workflow` | Разработка новой фичи | Сначала тесты, потом реализация | Текущий модуль |
| `code-review` | После написания кода | Ревью на качество, паттерны, безопасность | Diff изменений |

## Задачи, под которые планирую найти/собрать скилы

1. **Парсинг каталогов мебели** — автоматический сбор SKU, размеров, цен, фото с сайтов ретейлеров (Hoff, IKEA, Askona). Нужен скилл для structured scraping + нормализации данных.

2. **3D-визуализация из 2D** — конвертация плоского плана с размерами в 3D-превью расстановки мебели. Нужен скилл для работы с Three.js / Babylon.js.

3. **Переговоры с партнёрами (B2B outreach)** — шаблоны и структура для коммуникации с байерами мебельных сетей. Нужен скилл для составления партнёрских предложений.

4. **Prompt-инжиниринг для img2img** — систематический подход к подбору промптов для генерации интерьеров с сохранением геометрии. Нужен скилл с библиотекой проверенных промптов.

5. **Unit/integration тесты для AI-пайплайнов** — тестирование модулей, которые вызывают внешние AI API (моки, детерминированность). Нужен скилл специфично под AI-зависимый код.
